**2026-07-07 (Jake logs round 2): 0.2.4 was published but is DEAD ON ARRIVAL on every machine.** `render_health` → "Driver not found: …\.playwright\node\win32_x64\node.exe"; `install_browser` exits 1 in ~2ms ("Microsoft.Playwright assembly was found, but is missing required assets") because the installer itself needs the driver — no runtime recovery exists. Root cause: **single-file `dotnet publish` silently drops the Playwright driver's `node/win32_x64/node.exe`** (playwright-dotnet limitation; `package/**`+LICENSE survive, so the folder looks present), and `build-mcpb.ps1`'s only completeness check was the browser dir, not the driver. 0.2.3 had the same hole; the dev machine's own installed extension was equally broken (the release-checklist render_health was only ever run against dev builds, never a fresh .mcpb install). Two misleading breadcrumbs: the "Driver not found" path points two dirs up (`Claude Extensions\.playwright`) because it's Playwright's *last* (NuGet-layout) probe, and build-time browser install worked because it used the complete non-single-file dir. **0.2.5 fix:** build-mcpb.ps1 overlays `.playwright` from the non-single-file RID dir into staging + hard-fails if `bin\.playwright\node\win32_x64\node.exe` missing. Verify future respins via a real extension install, not a dev build.
