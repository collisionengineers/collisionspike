# a11y audit — `grid-native` (CELLWORK)

**Scope:** static WCAG-AA audit of the three runnable mockups in
`docs/plans/phase-ux-design-lab/directions-r2/grid-native/` — `index.html` (Inbox cockpit),
`queues.html` (grid), `case-detail.html` (record). Source-only review (rendered-DOM spot checks not run);
contrast computed from the frozen token values in each file's `:root`.

**Verdict: CONDITIONAL — does NOT pass AA as-is. Gate before this direction can win.**
**Score: 3.0 / 5.** Strong foundations (a global `:focus-visible`, real table/`<button>` semantics, a
fully AA-passing chip palette with shape-coding, the amber rule respected) undercut by a cluster of
concrete AA blockers: keyboard-inoperable view tabs and readiness items, a promised-but-absent
reduced-motion path, low-contrast `ink400` body text, and missing control state/labels.

---

## What passes (record, so the fixes don't regress it)

- **Chip palette is AA-clean.** Every `.h-*` foreground/background pair clears 4.5:1 — h-orange `#9A5212`
  on `#FBE7D6` ≈ 4.85, h-green `#14794A` on `#DDF3E4` ≈ 4.67 are the tightest; red/blue/indigo/purple/
  teal/pink all 5–6.8.
- **Amber rule honoured.** `.h-amber` is dark-on-amber (`#7A5A0A` on `#FAF0CE` ≈ 5.6:1). No white-on-amber
  anywhere. The CE blocker-red family (`#B01A33` on `#FCE4E6` ≈ 5.8) is well inside AA.
- **Colour is not the sole channel.** Status carries a glyph (`◆`/`▲`), provenance badges are shape-coded
  (`✔`/`●`/`▲`/`—`) **and** `title`-described, provider/channel chips carry text labels. Due-urgency is
  echoed in the adjacent time text, not bar-colour alone.
- **VRM plate** `#1C2230` on `#FFD400` ≈ 10.8:1.
- **Focus is globally visible:** `:focus-visible{outline:2px solid var(--acc);outline-offset:2px}`; the
  accent `#5043E6` gives ≈6.4:1 non-text contrast against white surfaces (clears the 3:1 SC 1.4.11 floor).
- Real semantics where it counts: `queues.html` uses a true `<table><thead><th><tbody>`; most actions are
  `<button>`/`<a href>`; `lang="en"` on all three; viewport allows zoom; the submit modal has
  `role="dialog"` + `aria-modal="true"` + `aria-label` + Esc-to-close.

## Findings (located, by severity)

### CRITICAL — AA-blocking (gate on these)

1. **Queues view tabs are not keyboard operable.** `queues.html:230–234` — the Not ready / Review / Held /
   Ready tabs are `<a class="vtab" data-view=…>` with **no `href`**. An anchor without `href` is not
   focusable and not Enter/Space-activatable; the whole tab set is mouse-only (SC 2.1.1). No tab semantics
   either — no `role="tablist"`/`role="tab"`/`aria-selected` (SC 4.1.2). → **stitch-prototyper.**
2. **Case-detail readiness items are mouse-only.** `case-detail.html:498–502` — `.rk.fail` rows are
   `<div onclick="deepLink(...)">` with no `tabindex`, role, or key handler. They're the primary
   "jump to the blocking field" affordance and a keyboard user cannot reach or fire them (SC 2.1.1). →
   **stitch-prototyper.**
3. **No reduced-motion path — anywhere.** Zero `@media (prefers-reduced-motion: reduce)` in any of the
   three files, despite `direction.md:201`/`seed.md:308` promising one. Animations actually shipped:
   `rowclear` (collapses row height, `index.html`), `cellflash` (`case-detail.html`), the bulk-bar
   transform/opacity rise (`queues.html`), modal show. The lab a11y floor requires a reduced-motion path
   (SC 2.3.3). → **stitch-prototyper** (gate the keyframes) / **ui-visual-designer** (confirm intent).

### SERIOUS — contrast + state/label exposure

4. **`ink400` (`#9AA1B0`) ≈ 2.6:1 on white — used for meaningful text.** Fails 4.5:1 (and 3:1).
   Affected, non-decorative text: section rails `.rsec` "Workspace"/"Tables"/"Admin"
   (`index.html:41`), the filter hint `.fhint` "facets set each row's verb" (`queues.html:98`),
   provider keys `.prov-key` (`queues.html:137`), muted pipeline counts `.stage .n.mut` (`index.html:97`),
   windowed-tile captions `.tile.win .cap` (`index.html:146`). The grey rail count `#9A5212` admin label is
   fine; the `ink400` ones are not. → **ui-visual-designer.**
5. **`ink500` (`#6B7383`) dips below 4.5:1 on tinted fills.** On `--neu-bg #EBECEF` ≈ **3.95:1** — hits the
   rail `.rcount` pills (`index.html:50`) and any neutral count badge. On `--canvas #F5F6F8` ≈ **4.31:1** —
   the `.kbd` keycaps (10px, `index.html:57`). Both are small text → fail. → **ui-visual-designer.**
6. **Toggle/selected controls don't expose state to AT (SC 4.1.2).** `.facet` chips toggle via a visual
   ✓-box but carry no `aria-pressed` (`queues.html:255–259`); `.vtab`/`.tab` "on" state is colour/border
   only, no `aria-selected`/`aria-current` (`queues.html:230`, `case-detail.html:298`); the Grid/Kanban/
   Calendar `.vb-btn` switcher's active item has no `aria-pressed` and no group semantics
   (`queues.html:238–247`). → **stitch-prototyper.**
7. **Unlabeled controls / icon-only links without accessible names (SC 4.1.2 / 2.4.4).**
   - Row checkboxes `.rsel` have no label (`queues.html:282` et al.) — AT announces a bare "checkbox".
   - Expand links: only row 1 has `title="Expand record"`; rows 2–8 are a bare `⤢` glyph anchor with no
     accessible name (`queues.html:295,307,319,…`).
   - Modal close `✕` `.mclose` has no `aria-label` (`case-detail.html:523`).
   - Chaser `<label>Channel</label>`/`<label>Template</label>` aren't tied to their `<select>`s (no `for`/
     no wrapping) (`case-detail.html:444–447`); the `.vb-search` filter input
     (`queues.html:242`), evidence role `<select>`s, and notes/IBA `<textarea>`s rely on placeholder/
     adjacent text only. → **stitch-prototyper.**
8. **Landmark + heading gaps.** No `<main>` in `index.html` or `queues.html` (content is `<div class="main">`;
   only `case-detail.html:296` has one). `case-detail.html` has **no `<h1>`** — first heading is the modal
   `<h2>` (broken hierarchy, SC 1.3.1/2.4.6). Panel titles across all three are styled `<span>`s, so there
   is no `<h2>` section structure to navigate by heading. The two `<nav>`s (rail + bottom) share no
   distinguishing `aria-label`. → **stitch-prototyper.**

### MODERATE

9. **Touch / target size below the 44px lab floor; some below the 24px WCAG 2.2 AA minimum (SC 2.5.8).**
   Row checkboxes render 15×15 (`queues.html:121`) and `.mini-btn` ≈ 20px tall (`index.html:121`) are
   **under 24px**; `.facet` ≈20px, `.use` ≈26px, `.refresh` 28px, `.abtn` ≈32px, `.tab`/`.vtab` ≈38–40px
   are all under the 44px floor. Mobile bottom-nav (58px) is the one comfortably-sized control. →
   **mobile-ux-designer.**
10. **Hidden bulk bar keeps its buttons in the tab order.** `.bulk` hides via `opacity:0;pointer-events:none`
    (not `display`/`visibility`), so its Hold/Release/Draft/Export buttons are still Tab-focusable while
    invisible (`queues.html:158`, SC 2.4.3/2.4.7). → **stitch-prototyper.**
11. **Modal has no focus management.** Focus isn't moved into the dialog on open, isn't trapped, and isn't
    restored to the trigger on close (`case-detail.html:520–547`, SC 2.4.3). Esc/scrim-close work. →
    **stitch-prototyper.**
12. **`aria-disabled="true"` link is still focusable and href-activatable** ("Engineer · reserved",
    `index.html:240`/`queues.html:215`) — `aria-disabled` doesn't block activation. Minor. →
    **stitch-prototyper.**

---

## Top fixes (do these and the verdict flips to PASS)

1. Make the **queues view tabs** real controls — `role="tab"` in a `role="tablist"`, focusable, with
   `aria-selected`; and make the **case-detail readiness `.rk.fail` items** `<button>`s (or
   `tabindex=0`+role+key handler). (Findings 1, 2)
2. Add a **`@media (prefers-reduced-motion: reduce)`** block to all three files that neutralises
   `rowclear`, `cellflash`, the bulk-bar rise, and modal transitions to instant state swaps. (Finding 3)
3. **Lift the two failing greys:** darken `ink400`→ keep it for decoration only and use `ink500`-or-darker
   for any real text; bump the `ink500`-on-`neu-bg` / `ink500`-on-`canvas` pairs to ≥4.5:1. (Findings 4, 5)
4. **Name and state the controls:** `aria-label` on row checkboxes, expand-glyph links, and the modal `✕`;
   `aria-pressed` on `.facet`/`.vb-btn`; associate the chaser/select/textarea labels. (Findings 6, 7)
5. **Structure:** add `<main>` to index/queues, an `<h1>` to case-detail, and `aria-label` the two navs;
   raise the smallest targets (15px checkbox, ~20px mini-btn) to ≥24px. (Findings 8, 9)
