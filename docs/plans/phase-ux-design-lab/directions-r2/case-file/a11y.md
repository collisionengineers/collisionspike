# A11y audit — "The Dossier" (case-file direction, R2)

Static WCAG-2.1/2.2-AA audit of the three runnable mockups in this folder:
`index.html` (Cockpit), `queues.html`, `case-detail.html`. Method: source read + contrast
computation (sRGB relative luminance) + semantics/keyboard/target inspection. No live DOM run —
findings are located by file + line/selector so the owning agent can act directly.

**Verdict: CONDITIONAL PASS — score 3 / 5.** A genuinely good a11y *foundation* (global
`:focus-visible` ring, reduced-motion gate on the stamp animation, shape **and** text labels rather
than colour-alone, real landmarks, `lang` set, most badges/plate pass contrast). But it carries
several **Level A** mechanical blockers (no headings, unlabeled/unassociated form controls) and a
handful of **AA contrast** misses (amber badges, faint text, red count on the dark rail). All fixes
are mechanical and do **not** threaten the visual concept — so this direction is *flagged, not
killed*: it can win convergence once the located fixes below land.

Severity: **A** = WCAG Level A failure (most serious) · **AA** = Level AA failure · **Med/Low** = quality.

---

## Cross-cutting failures (all three pages)

### 1. No document headings anywhere — **A** (1.3.1, 2.4.6)
Every page-, section-, and group-title is a `<span class="eyebrow">`; there is **no `<h1>` and no
`h1–h6` at all**. Screen-reader users get zero heading navigation and no page title in the a11y
tree. Section wrappers (`<section>` in index, panels in case-detail) have no accessible name.
- Fix: one `<h1>` per page ("Cockpit" / "Queues" / case id `CCPY26050`), `<h2>` on each
  `eyebrow` section header (the visual styling can stay — just change the tag, or add
  `role="heading" aria-level`). Owner: **stitch-prototyper**.

### 2. Unlabeled global search input — **A** (1.3.1, 3.3.2, 4.1.2)
The header search has a `placeholder` only, no label: `index.html:80`, `queues.html:68`,
`case-detail.html:84`. Placeholder is not an accessible name.
- Fix: add `aria-label="Search cases"` (or a visually-hidden `<label>`). Owner: **stitch-prototyper**.

### 3. Red "Review" count on the dark rail fails contrast — **AA** (1.4.3)
`<span class="railcount text-stamp">7</span>` on `bg-desk` — `#B23A2E` on `#2E2823` ≈ **2.45:1**
(needs 4.5). Located: `index.html:96`, `queues.html:80`, `case-detail.html:96`.
- Fix: use a lighter red token for on-dark counts (e.g. lift toward `#E0746A`, target ≥4.5), or a
  filled pill. Owner: **ui-visual-designer**.

### 4. Amber status/provenance badges fail contrast — **AA** (1.4.3)
`.s-held` and `.p-ai` are `#9A6B12` on `#F4E6C2` ≈ **3.78:1** (needs 4.5; the CE brand rule wants
8–9:1 dark-on-amber). Appears as "On hold"/"Held"/"Due soon" stamps and the "AI" source badge
(e.g. `index.html:125,206`, `queues.html:33,169-174`, `case-detail.html:34,190`).
Secondary: `.s-filed`/`.p-dvla` slate `#5E6B72` on `#E3E4DD` ≈ **4.29:1** (just under). `.s-review`
red ≈ **4.57:1** is a marginal pass.
- Fix: darken the amber ink to ~`#6E4A00`–`#5C3D00` (lands ~8:1) and the slate to ~`#465056`.
  Owner: **ui-visual-designer**.

### 5. `ink-faint` micro-copy fails contrast — **AA** (1.4.3)
`#9C8B6C` on cream ≈ **2.87:1**. Used at 10–11px for "throughput only…" (`index.html:241`),
"(not synced ⊘)" + "does not drive readiness" (`index.html:340`; `case-detail.html:338,340`).
- Fix: replace `text-ink-faint` with `text-ink-muted` (`#6B5C42`, ≈5.6:1) for any real text; keep
  `ink-faint` for borders/decoration only. Owner: **ui-visual-designer**.

### 6. Reduced-motion path is incomplete — **Low** (2.3.3 AAA / good practice)
The stamp `press` keyframe **is** gated (`@media (prefers-reduced-motion:reduce)`) — good. But
`html{scroll-behavior:smooth}` (`index.html:24`), the `scrollIntoView({behavior:'smooth'})` in
`deep()` (`case-detail.html:378`), and the `.fhi` highlight flash keyframe (`case-detail.html:71`)
are **not** gated.
- Fix: wrap those in the same reduced-motion query. Owner: **stitch-prototyper**.

### 7. Touch targets below the 44px lab target — **Low** (passes 2.5.8 AA 24px, misses 44px goal)
All ≥24px so no hard AA failure, but under the lab's 44px target: refresh button `w-8 h-8`=32px
(`*:83/70/86`), `.act` min-height 38px, `.facet` ~28px, `.fdrop` ~32px, `.qtab` ~36px, `.dtab`
~40px, `.fctl`/`.tinput` ~32–34px, rail items 42px / sub-items 38px.
- Fix: bump interactive min-height to 44px (rows in R4/queues already do). Owner: **mobile-ux-designer**.

## `index.html` (Cockpit) — specific

- Landmarks present (`<header>`, `<nav aria-label="Primary">`, `<main>`); menu/refresh buttons have
  `aria-label` — good.
- Decorative status glyphs (`✱ ▲ ◷ ⊘ ●`) all sit beside real text labels — **not colour-alone**, good.
- Sparkline SVG has `aria-label` — good.
- Inherits cross-cutting #1, #2, #3, #5, #6, #7.

## `queues.html` — specific

- **Tab pattern is incomplete — AA (4.1.2).** `role="tablist"`/`role="tab"` exist and `aria-selected`
  is maintained in JS (`:100-104, 214-219`), but there is **no `aria-controls`**, the rows container
  `#rows` is **not `role="tabpanel"`** (`:135`), and there's no arrow-key roving-tabindex. Buttons are
  Tab/Enter operable so it's usable, just not a conformant tabs widget.
- **Faux-labelled filters — A (3.3.2).** The `<select>` filters use a first `<option>` ("Provider ▾",
  "Status ▾"…) as a pseudo-label with no real label (`:114-117`); the search `#q` is placeholder-only
  (`:112`). Add `aria-label` to each.
- **Data grid has no table semantics — Med (1.3.1).** Rows are `<a>` + `<span>` cells with a `<div>`
  header (`:131-206`); column headers aren't associated. Rows themselves are focusable anchors with
  `min-height:44px` — **keyboard-operable, good**. Consider `role="table/row/cell"` + `scope`-style
  association, or a real `<table>`.
- **Duplicate flag is glyph/shape-only — Low.** `⊘` replaces "GB" on the plate (`plate()` `:187`) to
  mean "duplicate" with no text equivalent in the row. Add an SR-only "duplicate" string.
- Empty-state "clear all filters" is a real `<button>` — good.

## `case-detail.html` — specific

- **Field labels not programmatically associated — A (1.3.1, 3.3.2, 4.1.2).** Systemic: ~15 rows do
  `<label class="eyebrow">Work provider</label><select class="fctl">` with **no `for`/`id`** and no
  wrapping (`:181-202, 190-202`). Also the photo-classify `<select>` (`:237,242,247`) and the
  cover-sheet sequence `<input>` (`:356`) are unlabeled. (Radio/IBA/exclude inputs **are** wrapped by
  their `<label>` — those are fine.)
  - Fix: add `id` to each control + `for` on its label (or wrap). Owner: **stitch-prototyper**.
- **Modal focus management missing — A (2.4.3, 2.1.2) + 4.1.2.** `role="dialog" aria-modal="true"` and
  `Esc`-to-close are present (`:349, 383`) — good — but focus is **not moved into** the dialog on open
  nor **restored** on close, there's **no focus trap**, no `aria-labelledby` to the "Cover sheet" title,
  and the close `✕` button has **no accessible name** (`:352`).
- **Dialog is unreachable / dead trigger — Med (functional + 4.1.2).** The Submit button is `disabled`
  **and** carries `onclick="openSubmit()"` (`:140`) — a disabled button never fires click, so the modal
  can't open. Separately, the overflow **`⋯` tab** (`:171`) is a `<button>` inside the tablist that is
  **not `role="tab"`, has no handler, and no accessible name** (only a `title`) — a focusable dead
  control. Remove it or make it a real, named, operable control.
- **Divider tabs — same gaps as queues (AA, 4.1.2):** no `aria-controls`; panels are
  `<section data-panel hidden>` not `role="tabpanel"` and not `aria-labelledby` to their tab
  (`:165-172, 178-317`). Tabs are otherwise Tab/Enter operable and `aria-selected` is maintained.
- Evidence thumbnails use `alt=""` (`:236,241,246`) — defensible as decorative mocks, but real
  evidence photos should carry a description — Low.
- Provenance `src` badges, readiness checklist (`✓`/`✗` + text), and conflict `.flag` rows all pair
  shape/colour **with text** — **not colour-alone**, good.

## Strengths to preserve
- Global `*:focus-visible{outline:2px solid #B23A2E;outline-offset:2px}` — a visible focus ring on
  every interactive element (one caveat: 2px red on the dark rail/header is ~2.45:1 vs adjacent,
  under the 3:1 of 1.4.11 — give the on-dark focus ring a lighter colour or add an inner light halo).
- Reduced-motion gate on the headline stamp animation.
- Status/provenance/readiness are shape+text coded, not colour-alone.
- Real landmarks, `lang="en"`, zoomable viewport, primary body text ≈13:1, VRM plate ≈10.9:1.

## Top fixes (in priority order)
1. **(A)** Add a real `<h1>` + section `<h2>`s on every page.
2. **(A)** Associate every field `<label>` with its control (case-detail) + `aria-label` the search and
   filter selects (all pages).
3. **(A)** Modal: trap + move + restore focus, `aria-labelledby` the title, name the `✕` button; fix the
   disabled-button trigger and the dead `⋯` tab.
4. **(AA)** Lift contrast: amber ink `#9A6B12`→~`#5C3D00`, on-dark red count, slate `#5E6B72`, and swap
   `ink-faint` body text for `ink-muted`.
5. **(AA)** Complete the tabs widget: `aria-controls` + `role="tabpanel"`/`aria-labelledby` on
   queues and case-detail panels.

## Gating verdict for convergence
**Flagged — eligible to win conditionally.** No failure here is a visual-concept defect; every item is
mechanical markup/ARIA (→ **stitch-prototyper**) or a token-value contrast tweak (→ **ui-visual-designer**),
with target sizing (→ **mobile-ux-designer**). Clear fixes #1–#4 (the Level A set + the contrast set)
before this direction is declared the winner.
