# A11y audit — direction `pipeline-board` (BAY BOARD), round 2

**Auditor:** accessibility engineer · **Date:** 2026-06-25 · **Method:** static read of rendered
markup + CSS tokens + JS behaviour. Contrast computed from the design tokens (sRGB / WCAG 2.x).
**Scope:** `index.html` (Cockpit board), `queues.html` (tabbed grid), `case-detail.html` (case workspace).

**Verdict: GATED — does not pass WCAG-AA. A11y score 2.5 / 5.** Strong visual-accessibility
bones (a real `:focus-visible` ring, `prefers-reduced-motion` honoured, semantic `<table>`, real
`<button>`s for tabs/actions, icon+text badges so status is never colour-alone). But it carries
**multiple Level-A blockers** — primary rows/cards and a custom toggle are not keyboard-operable,
form inputs are unlabelled, the modal has no focus management — plus a **pervasive 1.4.3 contrast
miss** on the tertiary ink and a **white-on-amber** badge. Not eligible to win convergence until the
Level-A items below are cleared.

Severity key: **[BLOCKER]** Level-A / gating · **[HIGH]** AA fail, fix before port · **[MED]**
AA fail, narrow scope · **[LOW]** polish / AAA.

## Cross-cutting (all three pages)

### Contrast (1.4.3 / 1.4.11)
- **[HIGH] Tertiary ink `--ink3 #6E7C92` fails AA for normal text wherever it is used as real
  text.** Measured: 4.23:1 on white, 3.91:1 on `--well`, 3.74:1 on `--lane`/`morechip`,
  3.91:1 on `--lane-head` — all < 4.5:1, and the text is ~10–12.5px (not "large"). This is the
  single most widespread failure: it is the colour for email domains, `.meta`/timestamps, `.upd`
  "Updated 14:07", search placeholders, `.sysbadge`, `.morechip`, `.qtab .who`, `.seen "seen 14×"`,
  `.fact .fk` keys, `.aevt .ts`, `.facts-foot`. Darken `--ink3` to ≈ `#5B6678` (~4.6:1 on white,
  ~4.0 on washes) or restrict `--ink3` to ≥18px. → **ui-visual-designer**.
- **[HIGH] Disabled ink `--ink-dis #AAB3C2` = 2.11:1 on white used as live label text.** The
  **History tab** (`.tab.overflow`, `case-detail.html`) is a real navigation control rendered in
  `--ink-dis` — unreadable. Also the `🔒 locked` facts and the JSON "updates as fields edit" hint.
  Don't use `--ink-dis` for any text a user must read; give the overflow tab `--ink3`/`--ink2`.
  → **ui-visual-designer**.
- **[LOW] Focus ring** `--focus #5C8DF2` = 3.21:1 on white (passes the 3:1 non-text floor; the
  white inner ring keeps it visible on dark/coloured surfaces). OK — keep.

### Structure / landmarks (1.3.1 / 2.4.6)
- **[MED] No headings anywhere.** None of the three pages contain an `<h1>`–`<h6>`. Page titles
  ("Cockpit", "Queues"), the case identity, lane names and every `.sect-h` are styled `<span>`s.
  Screen-reader users get no heading outline to navigate by. Promote the page title to `<h1>` and
  section headers (`.sect-h`, lane `.lh-name`, sidebar `.sh`) to `<h2>/<h3>`. → **stitch-prototyper**.
- **[LOW] Duplicate `<nav>` landmarks unlabelled.** The rail has two `<nav>` blocks (Intake, Admin)
  with no `aria-label`; add `aria-label="Intake"` / `aria-label="Admin"`. → **stitch-prototyper**.

### Motion / zoom — PASS
- `@media (prefers-reduced-motion:reduce){*{animation:none!important;transition:none!important}}` is
  present on all three pages; the flash/lift/drag animations are disabled. **PASS** (2.3.3 / 2.2.2).
- `lang="en"`, `<title>`, and a zoomable viewport (no `maximum-scale`/`user-scalable=no`) on all
  three. **PASS** (3.1.1 / 1.4.4).

### Touch targets (2.5.8 AA = 24px; brief target = 44px)
- **[MED] Several controls below the 24px AA floor, most below the 44px brief target.** `.xbtn`
  inbox actions ≈ 22px tall, image-reorder `▲▼` buttons 22×22, the photo `.switch` 30×17. Chips
  (`.fchip`, `.fac`, `.pickbtn`) ≈ 24–26px and `.iconbtn` 32×32 clear 24 but miss 44. Bump the
  small action buttons/switch to ≥24px now and ≥44px for the production port. → **mobile-ux-designer**.

## `index.html` — Cockpit board

- **[BLOCKER] Job cards are not keyboard-operable (2.1.1).** `<article class="jc"
  onclick="openCase()">` has no `tabindex`, `role="button"`, or keydown handler — a keyboard user
  cannot focus a card or open a case. The `[ ` `]` "move card" shortcut only acts on the pre-set
  `.jc.sel`, and the click handler that should set `.sel` (lines 563–565) is a no-op, so selection
  can never move by keyboard either. Make cards `role="button" tabindex="0"`, open on Enter/Space,
  and move `.sel` with arrow/Tab focus. → **stitch-prototyper**.
- **[BLOCKER] Drag-to-advance has no robust keyboard path.** Status changes by drag
  (`draggable="true"`). The `[ ]` alternative exists but is dead without keyboard selection (above),
  so the core "advance a case" action is mouse-only. Wire the keyboard move to the focused card.
  → **stitch-prototyper**.
- **[HIGH] White-on-amber WIP badge (1.4.3) — the exact rule to avoid.** `.wip{color:#fff;
  background:var(--key)}` on the **Held** lane = white on `--held #D6912A` = **2.64:1 (fail)**.
  Other lanes also fail as ~12px text: Parsing 3.42, Ready 3.30, New 4.36 (only Submitted 4.53
  passes). Note the `.due.attn` pills do it correctly (dark `--held-ink` on `--held-wash` = 4.70) —
  apply the same dark-on-amber treatment to the WIP badge instead of white. → **ui-visual-designer**.
- **[MED] Hot Review lane header** white `.lh-name` on `--review #E04A36` = **4.04:1** — under 4.5
  for the 13px (non-large) lane title. Nudge `--review` slightly darker for the hot header, or use a
  larger/heavier title. → **ui-visual-designer**.
- **[MED] Funnel ribbon is colour + hover-title only (1.1.1 / 1.4.1).** `.ribbon` segment counts
  ("New 3", "Held 14"…) live only in `title=` on non-focusable `<span>`s; not exposed to keyboard or
  reliably to SR. Render the counts as text or give the ribbon an `aria-label`/visually-hidden legend.
  → **stitch-prototyper**.
- **[LOW] Readiness ticks `F·I·A·C` are colour-only.** `.tick.ok` (green) vs `.tick.no`/empty (grey)
  encode per-gate pass/fail with no text or shape difference. Secondary indicator (the badge+verb
  carry the real state) but add `aria-label`/title per tick or a shape cue. → **ui-visual-designer**.
- **[LOW] `.morechip` ("+4 more…") looks actionable but is a non-focusable `<div>`** (also not wired).
- **PASS:** `<aside>/<header>/<main>` landmarks; search input has `aria-label`; inbox
  Confirm/Reclassify/Mailbox are real focusable `<button>`s; `.iconbtn`s have `title`; status badges
  pair glyph **and** text label (not colour-alone).

## `queues.html` — tabbed grid

- **[BLOCKER] Grid rows are not keyboard-operable (2.1.1).** `<tr onclick="location.href=…">` with no
  `tabindex`/`role`/keydown. The rail hints "↵ open case" but nothing implements it and rows cannot
  receive focus — the entire queue is mouse-only. Make rows focusable + Enter-activates (or put a
  real link in the first cell). → **stitch-prototyper**.
- **[HIGH] Filter `<select>`s have no programmatic label (1.3.1 / 4.1.2).** Provider / Status /
  Channel / Age selects sit next to a `.fil` `<span>` of text but it is not a `<label for>`; SR
  announces a nameless combobox. Add `aria-label` (or wrap in `<label>`). → **stitch-prototyper**.
- **[MED] Tabs lack tab semantics / selected state (4.1.2).** `.qtab` are real `<button>`s (good) but
  there is no `role="tablist"`/`role="tab"`, and the active tab is class-only (`.on`) with no
  `aria-selected`/`aria-current`, so SR users can't tell which queue is active. Add roles +
  `aria-selected` and tie to the grid via `aria-controls`. → **stitch-prototyper**.
- **[MED] "showing 8 of 8" count is not a live region.** `#nofm` updates on filter silently; wrap in
  `aria-live="polite"`. Facet toggles also lack `aria-pressed`. → **stitch-prototyper**.
- **[LOW] `<th>` lack `scope="col"`** (minor; semantics otherwise correct).
- **[LOW] Channel cell** `✉`/`⌘` carries meaning only via `title` on a `<span>`.
- **PASS:** proper `<table><thead><th>` structure; `qsearch` has `aria-label`; facets/empty-state
  "clear filters" are focusable; rows are tall (~40px).

## `case-detail.html` — case workspace

- **[BLOCKER] Editable fields are unlabelled (1.3.1 / 4.1.2).** Claimant name, Mileage, Mileage-unit,
  **VAT status** (`#vatSel`), Circumstances, IBA reason, per-photo Role selects, the chaser
  Channel/Template selects and the Case/PO sequence input all lack `<label for>`/`aria-label`. The
  visible `.fl` text is an unassociated sibling `<span>`. Every required intake field is a nameless
  control to SR. Associate each via `<label for>` or `aria-label`; add `aria-required`/`aria-invalid`
  to the VAT field and link `#vatErr` via `aria-describedby`. → **stitch-prototyper**.
- **[BLOCKER] Photo-exclude toggle is keyboard-inoperable + has no role (2.1.1 / 4.1.2).**
  `<span class="switch" data-sw>` flips on click only — not focusable, no `role="switch"`,
  no `aria-checked`, and "Exclude (reflection)" is not associated. This is the domain-critical
  person-reflection exclusion; a keyboard user cannot set it. Rebuild as a `<button role="switch"
  aria-checked>` (or a real `<input type="checkbox">`) with an associated label. → **stitch-prototyper**.
- **[BLOCKER] Submit modal: no focus management (2.4.3 / 4.1.2).** `role="dialog" aria-modal="true"`
  is set (good) and Esc/scrim-click close (good), but on open focus is **not** moved into the dialog,
  there is **no focus trap** (Tab escapes to the page behind), focus is **not restored** on close, and
  the close control `<button class="x">×</button>` has **no accessible name**. Move focus to the
  dialog on open, trap Tab within it, restore on close, and add `aria-label="Close"` to `×`.
  → **stitch-prototyper**.
- **[HIGH] Sidebar readiness checklist links are not keyboard-operable (2.1.1).** `.chk[data-go]`
  rows ("Fix ›", "Fields · 8 ›", "Evidence ›") are `<div>`s with click handlers — they look like
  links/buttons but can't be focused or activated by keyboard. The tabs provide a redundant keyboard
  path so it's not a total dead-end, but make these `<button>`s. → **stitch-prototyper**.
- **[MED] Tab/panel ARIA missing.** `.tab` are real `<button>`s and panes are `display:none` when
  hidden (good — SR skips them), but no `role="tab"`/`tabpanel`, no `aria-selected`, no
  `aria-controls`/`aria-labelledby`. → **stitch-prototyper**.
- **[MED] Icon-only reorder buttons unnamed (4.1.2).** Image-order `▲`/`▼` are real `<button>`s with
  no text/`aria-label` (SR reads "up-pointing triangle"); also 22×22px (< 24). Add
  `aria-label="Move up/down"`. → **stitch-prototyper** (label) / **mobile-ux-designer** (size).
- **[MED] Required-field state is colour-reinforced but the text error is good.** `.frow.req` shows a
  red border + tinted bg; the `.ferr` text "Required field — cannot submit until set" is real text
  (good), but state isn't exposed via `aria-invalid`/`aria-required` (see field-label blocker).
- **PASS:** action buttons (Add evidence / Merge / Hold / Submit / Delete) are real `<button>`s with
  text labels and correct `disabled` gating; Delete/Download pair emoji **with** a text word;
  `<details>/<summary>` Live-JSON disclosure is natively keyboard-operable; provenance pills pair
  glyph **and** text (PDF/AI/Manual/None) so provenance is not colour-alone; status badges/`.due`
  pills meet contrast (review-ink/review-wash 5.37, held-ink/held-wash 4.70).

## Gating summary

**6 Level-A blockers** (keyboard operability of board cards, grid rows, the readiness-checklist
links, the photo-exclude switch; modal focus management; unlabelled form fields) plus **2 high AA
contrast misses** (`--ink3` body text, white-on-amber WIP). **Convergence verdict: BLOCKED.** Clear
all `[BLOCKER]` items and the two `[HIGH]` contrast fixes before this direction is eligible to win.
Routing: contrast → **ui-visual-designer**; markup/ARIA/keyboard → **stitch-prototyper**; target
sizing → **mobile-ux-designer**. Rubric/scoring owners: **ux-architect** (IA) / **design-critic**
(this is the a11y dimension feed).
