# Accessibility audit ‚Äî `directions-r2/workbench`

**Scope:** static WCAG 2.1/2.2-AA audit of the three runnable mockups in this folder
(`index.html` Cockpit, `queues.html`, `case-detail.html`). Source-read audit (markup + computed
contrast), not a rendered-DOM/axe pass ‚Äî re-confirm against the live render before any production sign-off.

**Verdict:** CONDITIONAL ‚Äî good accessible bones (real landmarks, real links/buttons, visible focus
ring, reduced-motion path, shape+colour status coding), but **not yet AA-clean**. Blocking gaps are
markup-semantic (form inputs not programmatically labelled; the queues "grid" is not a table; no `h1`)
plus a cluster of **marginal contrast** pairs on amber/gold text and on chrome-tinted backgrounds.

**Score: 3 / 5** (advisory).

## Per-page scorecards

### `index.html` ‚Äî Cockpit ‚Äî 3/5
- **Focus:** PASS ‚Äî `.focusring` (2px halo on `:focus-visible`) on rail links, buttons, card links,
  triage actions; row links fall back to the UA outline (not reset globally). Visible focus exists
  everywhere interactive.
- **Landmarks:** partial ‚Äî `nav` (rail) + `main` + `aside#explorer` present, single `main`. But TWO
  `<nav>` (rail `:60` + phone bottom nav `:445`) share no `aria-label` ‚Üí indistinguishable to AT
  (2.4.1). No `h1` on the page ‚Äî first heading is `h2` "Pipeline" (`:158`). The triage column titles
  "Receiving work / Queries / Other" are `<span class="sc">`, not headings.
- **Names on controls:** rail links get a name from `title=` (OK but `title`-only is weak); the inner
  SVGs are not `aria-hidden`, so the AT tree carries redundant graphics. The "+" new-tab button `:140`
  has `title="New tab"` (OK). The search toggle `:143` exposes visible text ‚Äî good.
- **Colour-only:** statuses are shape+colour coded (`‚úï ‚óê ‚ñ≥ ‚è∏ ‚úì ‚óÜ`) ‚Äî good. The blue "unsaved review"
  dot `:92` has `title`; the same dot on the tab `:132` has no title (colour-only) ‚Äî minor.
- **Reduced motion:** PASS ‚Äî `@media (prefers-reduced-motion:reduce)` kills `fadein`/transitions `:48`.
- **Targets:** rail items 36√ó36 (`w-9 h-9`) and triage action buttons `h-6` (24px) clear the 2.5.8 AA
  24px floor but miss the 44px target the brief asks for.

### `queues.html` ‚Äî 3/5
- **Tabs/segments:** the segmented selector (`:113‚Äì124`) and the explorer queue links are real
  `<button>`/`<a>` with `.focusring` ‚Äî keyboard operable. BUT no tab semantics: no
  `role="tablist"/tab"`, no `aria-selected`/`aria-pressed`. Active state is conveyed by `seg-on`
  (underline + colour) so it is not colour-alone visually, but a screen reader cannot tell which
  segment is active (4.1.2). Owner: **stitch-prototyper**.
- **Data grid is not a table (HIGH):** the queue is CSS-grid `<div class="qhead">` + `<div/a
  class="qrow">` (`:160‚Äì322`) with `<span>` cells. Column headers (VRM/Case-PO/Provider/Status/
  Outstanding/Ch/Age) are not programmatically associated with cells ‚Üí 1.3.1 relationships lost; a
  screen-reader user hears an undifferentiated run of spans. Add `role="table/row/columnheader/cell"`
  (or real `<table>`). Owner: **stitch-prototyper**.
- **Icon-only channel cell:** `‚úâ`/`WA` cells use `title=` on some rows (`:172` `title="email"`) but
  omit it on others (`:248`, `:257` bare `‚úâ`) ‚Äî inconsistent accessible text for the channel.
- **Search input `:133`** has placeholder only, no `<label>`/`aria-label` ‚Üí no durable accessible name
  (3.3.2/4.1.2). Same on the cockpit/palette search boxes.
- **"+" tab button `:100`** has no `title`/label ‚Äî accessible name resolves to literal "+".
- **Targets:** facet chips `h-7` (28px), segment buttons `h-9` (36px) ‚Äî pass 24px, under 44px.
- Reduced-motion PASS; focus ring PASS.

### `case-detail.html` ‚Äî 3/5 (richest surface, most gaps)
- **Form labels (HIGH, pervasive):** the 12 field inputs/selects/textareas pair a `<label>` with an
  input as **siblings with no `for`/`id`** (e.g. `:195` Work provider, `:200` Claimant, `:205`,
  `:210`, `:222`, `:229` mileage, `:236` select, `:243` VAT select, `:257` textarea, `:272`, `:276`).
  No programmatic name ‚Üí 1.3.1 / 3.3.2 / 4.1.2 fail across the whole Fields pane. (The IBA radio `:384`
  and the photo "exclude" checkbox `:328` DO wrap their input ‚Äî those are fine.) Owner:
  **stitch-prototyper** (wrap input in the `<label>`, or add `for`/`id`).
- **Sub-tabs (Fields/Evidence/Address/Notes/Chasers `:120‚Äì124`)** and the terminal tabs `:481‚Äì483`:
  real buttons + `.focusring`, panes toggled by `display:none` (hidden panes correctly drop out of the
  AT tree) ‚Äî but no `role="tab"/tabpanel"`, no `aria-selected`, no `aria-controls`. The Evidence tab's
  red "needs attention" dot `:121` is colour-only (no text/title). Owner: **stitch-prototyper**.
- **"Keyboard-reorderable" claim is false:** the EVA photo-order list `:351‚Äì355` labels itself
  keyboard-reorderable but the `‚åÉ ‚åÑ` up/down affordances are inert `<span>`s, not buttons ‚Äî not
  focusable or operable. Either make them real buttons or drop the claim. Owner: **stitch-prototyper**.
- **Dialogs:** the submit modal `:534` and command palette `:572` move focus to an input on open and
  close on `Esc` (good), but have no `role="dialog"`/`aria-modal`, no accessible name, no focus trap,
  and leave the background operable (not `inert`/`aria-hidden`). 4.1.2 / focus-management gap. Owner:
  **stitch-prototyper**.
- **Disabled Submit/Download** use real `disabled` + explanatory `title` (good pattern); low-contrast

- **Decorative glyph noise:** provenance chips (`PDF ‚úì`, `AI ‚óè`), status glyphs and the `üîí`/`‚ú¶`/`‚§ì`
  characters are not `aria-hidden`; AT will read them literally. Wrap in `aria-hidden` + visually-hidden
  text. Owner: **stitch-prototyper**.
- **Targets:** inline resolver buttons are **`h-5` = 20px** ‚Äî *below* the 2.5.8 AA 24px floor
  (`:230` "use MOT"/"keep PDF", `:336` "re-include", `:261` "‚Üí Address tab", `:377‚Äì379` picks are

  **mobile-ux-designer** (raise the `h-5` controls to ‚â•24, ideally 44 hit-area via padding).
- Reduced-motion PASS (`flash`/`resolved-fade`/`fadein` all disabled `:49`); focus ring PASS.

## Top fixes (ordered by gating impact)

1. **Associate every Fields input with its label** (`case-detail.html` Fields pane) ‚Äî wrap or
   `for`/`id`. Highest-volume AA fail. ‚Üí stitch-prototyper.
2. **Give the queues table real table semantics** (`role=table/row/columnheader/cell` or `<table>`) so
   column‚Üícell relationships are programmatic. ‚Üí stitch-prototyper.
3. **Fix the contrast cluster:** darken `tfn` gold (3.57), lift `tx3` on `chrome1`/`chrome2`/`gutter`
   surfaces (3.88‚Äì4.41), use `accentp` for the selected-nav label on `asoft` (3.93), and stop using
   `txd` for meaningful gutter numbers/placeholders (2.1‚Äì2.3). ‚Üí ui-visual-designer.
4. **Add ARIA to the tab/segment patterns and to the two dialogs** (`role=tab/tablist/tabpanel` +
   `aria-selected`; `role=dialog`+`aria-modal`+name+focus-trap+background-inert). ‚Üí stitch-prototyper.
5. **Touch targets:** raise `h-5` (20px) inline buttons to ‚â•24px now, and grow rail items (36px) and
   the row/queue action buttons toward the 44px target. ‚Üí mobile-ux-designer.
6. **Name the icon-only controls** ("+" tab buttons, channel cells) and `aria-hidden` the decorative
   glyphs/SVGs; add an `h1` per page and `aria-label` to the duplicate `<nav>`s. ‚Üí stitch-prototyper.

**Strong positives to keep:** consistent `:focus-visible` halo, a genuine `prefers-reduced-motion`
path on all three pages, shape+colour status coding, real semantic `nav`/`main`/`aside`, keyboard
