# a11y audit ‚Äî directions-r2 / product-minimal ("QUIET")

**Scope:** static source audit of `index.html`, `queues.html`, `case-detail.html`.
**Method:** source read + computed WCAG contrast ratios (sRGB relative-luminance). No live DOM render.
**Standard:** WCAG 2.1 AA + the CE accessibility floor (focus visible, reduced motion, AA contrast, shape-AND-colour coding).
**Date:** 2026-06-25.

**Verdict: FAIL the AA gate.** Contrast and the focus indicator are strong, but core interactions on
all three pages are built on non-focusable `<span>`/`<div>` widgets, so a keyboard-only user cannot
complete the primary task (resolve VAT + confirm reg-visible ‚Üí enable *Submit to EVA*). No page has a
single heading element. **Score: 2.5 / 5.**

Severity: **BLOCKER** (AA fail that stops a core task) ¬∑ **MAJOR** (AA fail) ¬∑ **MINOR** (borderline / best-practice).

## index.html (Cockpit) ‚Äî score 3/5

- **MAJOR ‚Äî no headings.** Zero `<h1>‚Äì<h6>` in the document; `.topbar__title`, `.sec__label` are styled spans. Breaks 1.3.1 / 2.4.6 and heading-jump navigation.
- **MAJOR ‚Äî command palette is keyboard-dead.** `.cmdk` opens and focuses the input, but `.cmdk__item` entries are `<li>` with only `mouseenter` handlers ‚Äî no roles (`listbox`/`option`), no arrow-key move, no Enter-to-run. Selecting any command is mouse-only.
- **MAJOR ‚Äî `.cmdk` dialog has no focus trap / `aria-modal`.** Tab escapes to the page behind. (Esc-to-close ‚úì, focus-in-on-open ‚úì.)
- **MINOR ‚Äî j/k cursor ‚âÝ real focus.** `#chaseList` arrow/j-k move a visual `.is-cursor` class; DOM focus never moves, so AT users don't perceive the cursor and the `Enter`-opens path is invisible to them. The rows themselves are `<a href>` and are independently tab-operable ‚úì.
- **MINOR ‚Äî colour-alone blocker.** `.nav__count.is-blocker` signals "blocker" only by turning the count red; no glyph/text (the ruler/tiles correctly pair `‚ñÝ`/`‚ñ≤` glyphs).
- **MINOR ‚Äî decorative nav glyphs not hidden.** `.nav__ico`/`.ci`/`.glyph-sq` unicode marks aren't `aria-hidden`; SR reads stray symbol names. (Text labels are present, so not blocking.)
- **MINOR ‚Äî icon-only buttons.** `.iconbtn` Refresh has `title` ‚úì; density buttons have `title` on this page ‚úì but no `aria-pressed` for the active state.
- PASS: landmarks (`aside`/`nav`/`header`/`main`), `lang`, zoomable viewport, global `:focus-visible` 4px accent halo on every focusable element, partial reduced-motion (`drainRoll` only ‚Äî `tickIn` not covered), chase rows `min-height:44px`.

## queues.html (Queues) ‚Äî score 2/5

- **BLOCKER ‚Äî inline edit not keyboard reachable.** `.live` cells are bare `<span>` (no `tabindex`/role) nested **inside** the `<a class="grow gdata" href>` row. Tab lands on the row link (navigates to case-detail); the edit affordance is unreachable. The `e`-to-edit shortcut only fires off the visual-only j/k cursor, which never takes real focus.
- **MAJOR ‚Äî nested interactive.** Click-to-edit control inside an anchor is invalid and ambiguous to AT (activating the row = follow link; the edit is hidden from the a11y tree).
- **MAJOR ‚Äî broken tab pattern.** `.partition` has `role="tablist"` and `role="tab"` buttons, but: no `aria-controls`, no `tabpanel`, no roving-tabindex/arrow-key navigation, and `aria-selected` is set only on the active tab at load (3 tabs ship with no `aria-selected`).
- **MAJOR ‚Äî unnamed icon-only buttons.** Density buttons (`‚ñ¶ ‚ñ§ ‚ñ¨`) have **no `title` and no text** on this page ‚Üí accessible name is the glyph only. No `aria-pressed`.
- **MAJOR ‚Äî no reduced-motion path.** This page defines `tickIn` (savetick) + transitions but has **no** `@media (prefers-reduced-motion)` block at all.
- **MAJOR ‚Äî no headings** (same as index).
- **MINOR ‚Äî filter state not exposed.** `.fchip` reason filters toggle `is-on` visually but lack `aria-pressed`; `.fdrop` dropdowns lack `aria-haspopup`/`aria-expanded`.
- **MINOR ‚Äî info via `title` on non-interactive `<div>`.** `.gchan` channel (`‚úâ`/`‚åæ`) and `.dupflag` `‚äò` convey meaning only through a glyph + `title`; SR support for `title` on a `div` is unreliable.
- **MINOR ‚Äî `#qsearch` input** relies on placeholder as its only accessible name (no `<label>`/`aria-label`).
- PASS: badges pair text+shape-coded dot (square=danger/warn, circle=ok/neutral), severity always paired with a due-pill number (not colour-alone), grid rows `min-height:44px`, focus ring.

## case-detail.html (Case) ‚Äî score 2.5/5

- **BLOCKER ‚Äî primary task not keyboard-completable.** *Submit to EVA* is gated on two readiness items, and neither is keyboard-operable:
  - VAT fix ‚Üí `.live #vatVal` is a non-focusable `<span>`; the only pointer-free path is the readiness deep-link `.rdz__item` which is a `<div>` (not focusable either).
  - Reg-visible ‚Üí `.sw #regSw` toggle is a `<span>` with a click handler: not focusable, no `role="switch"`, no `aria-checked`, label text is an unassociated sibling.
- **MAJOR ‚Äî broken tab pattern.** `.tabs role="tablist"` but `.tab` buttons have **no `role="tab"`**, no `aria-selected`, no `aria-controls`; `.tabpanel` sections have no `role="tabpanel"`/`aria-labelledby`; no arrow-key roving.
- **MAJOR ‚Äî fake-disabled actions.** `#submitBtn`/`#dlBtn` use a visual `.is-disabled` class with `title` only ‚Äî no `disabled` or `aria-disabled`. They stay in tab order and read as enabled to AT; clicking is a silent JS no-op.
- **MAJOR ‚Äî overflow menu inoperable.** `#ovBtn` is labelled ‚úì but lacks `aria-haspopup`/`aria-expanded`; `.ovi` items are `<div>` (no `menuitem` role, not focusable) ‚Üí mouse-only.
- **MAJOR ‚Äî modal focus management.** `#submitModal role="dialog"` has no `aria-modal`, focus is not moved into it on open, not trapped, not restored on close; background not inert. (Esc-closes ‚úì.)
- **MAJOR ‚Äî readiness changes not announced.** `.msgbar` / readiness panel update dynamically with no `role="alert"`/`aria-live`.
- **MAJOR ‚Äî no headings** (`.cluster__h`, `.subh`, `.panel__h` are spans/divs).
- **MINOR ‚Äî unlabeled controls:** thumb role `<select>` (no label), order `‚Üë/‚Üì` `[data-mv]` buttons (glyph-only name, no "move photo up/down"), `.modal__close ‚úï` (no `aria-label`), `#seqInput` (sibling "Sequence" text unassociated).
- PASS: **comprehensive reduced-motion** (`*{animation:none!important;transition:none!important}`), focus ring, badge text+shape coding, VAT shown as text "‚úó required" not colour-alone, contrast on tint message bars.

## Top fixes (priority order)

1. **Make the gating widgets real, focusable controls** (BLOCKER). Inline-edit `.live`, the `.sw` toggles, `.ovi` menu items, and `.cmdk` items must be `<button>`/`<input>`/`role="switch"`+`tabindex` with `aria-checked`/Enter-Space activation ‚Äî so VAT and reg-visible (and Submit) are reachable by keyboard. ‚Üí **stitch-prototyper**
2. **Un-nest interactive from the row anchors on queues** ‚Äî don't put edit controls inside `<a href>`; split row-navigate from cell-edit. ‚Üí **stitch-prototyper**
3. **Add a heading outline** (`<h1>` page title + `<h2>` section/cluster headings) on all three pages. ‚Üí **stitch-prototyper**
4. **Fix the ARIA tab/menu/dialog patterns** ‚Äî `role="tab"`+`aria-selected`+`aria-controls`+roving arrows; `tabpanel`+`aria-labelledby`; `aria-modal`+focus trap/restore on modal & cmdk; `aria-pressed` on density/fchip toggles; real `disabled`/`aria-disabled` on Submit/Download; `aria-live`/`role="alert"` on the readiness messagebar. ‚Üí **stitch-prototyper**
5. **Name every icon-only control** ‚Äî queues density buttons, order `‚Üë/‚Üì`, modal `‚úï`, thumb `<select>`, channel/dupflag glyphs (move meaning off `title`-on-`div`). ‚Üí **stitch-prototyper**
6. **Add the missing reduced-motion block to `queues.html`** (mirror case-detail's global rule). ‚Üí **stitch-prototyper**
7. **Contrast tweaks** ‚Äî darken the `.notconn`/`.box-link`/`.fact--gated` status text off `--ink-disabled` (‚â•4.5 on its surface); lift `ok`-on-`ok-tint` (4.45) and `ink-3`-on-`subtle-2` (4.21) over 4.5; add a glyph/text to the `.nav__count.is-blocker` so the blocker isn't colour-alone. ‚Üí **ui-visual-designer**
8. **Touch targets** ‚Äî bring `.olrow__mv` to ‚â•24 (target 44) and pad the small chip/icon buttons toward 44. ‚Üí **mobile-ux-designer**

**Production-port note (‚Üí fluent-codeapp-designer):** porting these to Fluent v9 primitives (`Tab`/`TabList`,
`Switch`, `Menu`, `Dialog`, `Button disabled`) resolves fixes 1, 4, 5 for free ‚Äî the prototype's
hand-rolled `span`/`div` widgets are the root cause of the keyboard failures.
