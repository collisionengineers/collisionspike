# A11y audit — `split-triage` (Cold Slate Reader) — WCAG-AA

Static audit of the three mockups in this folder against the WCAG-AA floor (contrast, visible
focus, semantic structure/landmarks, ARIA/labels, keyboard operability, touch targets, reduced
motion). Source-level audit; contrast ratios computed from the frozen palette.

- **Verdict:** GATED — not convergence-ready. Real keyboard operability and a visible focus ring are
  in place, but four AA blockers recur across all three pages.
- **A11y score:** **2.5 / 5**
- **Files:** `index.html` (cockpit/inbox), `queues.html`, `case-detail.html`

Severity key: **[B] blocker** (AA fail, gates convergence) · **[M] major** · **[m] minor/advisory**.

---

## What passes (credit where due)

- **Visible focus ring** on every focusable: `a/button/[tabindex]:focus-visible` → `0 0 0 2px #fff,
  0 0 0 4px var(--accent)` (2px white + 2px iris). Meets 2.4.7 / 2.4.11.
- **Not colour-alone** — status badges pair colour + a shape glyph + a word (`◣ Review`, `◣ Conflict`);
  provenance pairs `✔/●/▲` + `PDF/AI/DVLA`; readiness uses `✔/✗` + label; catdots always sit next to a
  text label. Meets 1.4.1.
- **Passing contrast:** VRM plate (11.8:1), primary buttons white-on-iris (5.4:1), iris links on
  white/canvas (5.4 / 4.7:1), `muted` (5.98:1) and `ink700` (10.2:1) body text, `b-ready` (5.4:1).
- **Landmarks:** `<header>`, `<nav aria-label="Primary">`, `<nav aria-label="Sections">`, `<aside>`.
  `lang="en"`, per-page `<title>`, zoomable viewport (no `maximum-scale`).
- **Keyboard model on lists:** J/K cursor, Enter/E open, ⌘K palette, Esc, `[`/`]` pane toggle.
- **Real tablist** in `case-detail` (`role="tablist"`/`role="tab"`/`aria-selected`) + 1–5 jump keys.
- **Reduced motion** honoured for the cockpit morph in `index.html`.

## Blockers (AA — gate convergence)

### [B1] No heading structure and no `<main>` — all three pages
There is **not a single `<h1>`–`<h6>`** anywhere, and no `<main>` landmark. Section titles are
`<span class="eyebrow">`/`<div>`. Screen-reader users get no document outline, no "skip to main", no
page-level heading. Violates 1.3.1, 2.4.1, 2.4.6.
- Locations: every page. The reading `<section>` (`index.html` L281, `queues.html` L156,
  `case-detail.html` L188) should be `<main>`; eyebrow titles like "Chase next" (L374), "Readiness
  checklist" (L446) should be real headings.
- Owner: **stitch-prototyper**.

### [B2] Form controls with no programmatic label — pervasive (`case-detail.html`, `queues.html`, `index.html`)
Visible text exists but is **not associated** with the control (`<label for>` / `aria-labelledby` /
`aria-label` all absent). Each control's accessible name is empty or relies on a placeholder. Violates
1.3.1, 3.3.2, 4.1.2.
- `case-detail.html` EVA field inputs 1–12 (L261–285): `.lbl` span is adjacent, not linked. VAT
  `<select>` (L271) similarly. Photo role `<select>` (L328/336). `#ibaReason` textarea (L383, placeholder
  only). Notes textarea (L392). Chaser `<select>`/`<textarea>` (L412–417). Submit-modal `Principal/Year/
  Sequence` inputs (L491–493) have `<label>` text but no `for`/`id` pairing.
- `queues.html` filter `<select>`s (L169–172, name carried only by first `<option>`), `#gridSearch`
  (L175, placeholder only).
- `index.html` `#cmdkInput` (L541) and `queues`/`case-detail` `#cmdkInput` — placeholder only.
- Good counter-examples already in the file: exclude/address/IBA checkboxes are wrapped in `<label>`
  (copy that pattern everywhere).
- Owner: **stitch-prototyper** (markup) — wrap or add `aria-label`/`for`.

### [B3] Queue selector is mouse-only — `queues.html`
The left-spine queue items and saved views are `<div class="qitem" aria-selected="…">` with **no
`tabindex` and no `role`** (L136–145). They have JS `click` handlers only, so they are **not reachable
or operable by keyboard**, and `aria-selected` on a roleless `<div>` is invalid ARIA (no exposed
selected state). Violates 2.1.1, 4.1.2. The documented "switch queue" path has no keyboard route to
these items.
- Owner: **stitch-prototyper** — make them `<button>`s in a `role="tablist"`/`listbox`, or add
  `tabindex="0"` + `role` + key handling.

### [B4] Contrast fails on `faint` text + amber `text-held` — all three pages
- `faint` `#828A98` is the most widespread fail: **3.48:1 on white, 3.21 on sunken, 3.02 on canvas**
  (need 4.5). Used for timestamps, secondary metadata ("instruction", "query · read"), counts, "…2
  more", `spseg` future states, etc. — at 10–11px (never "large text"). Examples: `index.html`
  L181/183/193, `queues.html` L177/201, `case-detail.html` L214/461/470.
- Amber `text-held` `#C07A12`: **3.48:1 on white** (queues spine "Held 14" L138; cockpit context pane
  L518) and **3.11:1 on the `#FBF1DF` funnel tile** (`index.html` L316). This is the brand's
  "amber-never-white-on-amber" spirit inverted — it's a too-light amber on a near-white/amber ground.
- Owner: **ui-visual-designer** — darken `faint` to ≈`#6B7280` (→ ~4.6:1) and `text-held` to the
  brand's `#8A5A00`-ish dark amber on light grounds.

## Major

### [M1] Badge / due-pill text sits on the 4.5 line — all pages
Computed: `b-review` & `due-hot` `#D92D32`/`#FDECEC` = **4.21:1 (fail)**; `b-conflict` = 4.52;
`b-held`/`due-warn` = 4.50; `text-review` on canvas = 4.16 (fail). The two red ones (4.21) are sub-AA
at 10.5px/600. Nudge the red tints darker or text darker.
- Owner: **ui-visual-designer**.

### [M2] `aria-selected` on non-tab roles — `queues.html`
The segmented `.seg` controls are `<button aria-selected>` (L160–163) with **no `role="tab"`** —
`aria-selected` is not valid on a plain button. Same anti-pattern as B3. Use `role="tab"` in a
`role="tablist"`, or switch to `aria-pressed`.
- Owner: **stitch-prototyper**.

### [M3] Tabpanels not associated with tabs — `case-detail.html`
`role="tablist"`/`role="tab"` present (L244–249) but tabs have no `id`/`aria-controls`, and the
`.tabpanel` divs (L257, L310, …) have **no `role="tabpanel"`/`aria-labelledby`**. The tab/panel
relationship is invisible to AT, and there's no arrow-key roving (APG). Add the wiring.
- Owner: **stitch-prototyper**.

### [M4] Overlays are not dialogs — all pages
`#cmdk`, `#help`, `#submitModal`, and the `#ovf` overflow menu are `hidden` divs with **no
`role="dialog"`/`aria-modal`, no focus trap, and no focus return** to the trigger on close. Background
stays focusable behind the scrim. Violates 4.1.2 / 2.4.3 expectations for modal surfaces.
- Owner: **stitch-prototyper** (semantics) — `role="dialog" aria-modal="true"`, trap + restore focus.

### [M5] List rows have no role/name — `index.html`, `queues.html`
Inbox rows (`.mrow`, tabindex injected by JS, L177+) and grid rows (`.grow`, L198+) are focusable
`<div>`s with **no `role` and no accessible name**; AT announces a generic group. Also the
cursor model (J/K `data-cursor`) is decoupled from DOM focus, so Tab-focusing a row then pressing
Enter acts on the *cursor* row, not the focused one. Give rows `role="button"`/`role="row"` + an
`aria-label` (VRM + case + status), and sync cursor with focus.
- Owner: **stitch-prototyper**.

## Minor / advisory

- **[m] Touch targets < 44px** (2.5.5, AAA but on the brief): facet chips 24px (`queues` L183–187),
  filter selects 28px, EVA `.ctrl` inputs 28px, `.actbtn` 30px, tabs 34px, icon buttons `h-7 w-7`
  28px, header refresh ~16px (`index` L123), `kbd` close buttons 18px. Most clear WCAG 2.2's 24px
  *minimum* (2.5.8) but miss the 44px target. Owner: **mobile-ux-designer**.
- **[m] Reduced motion only partial:** the `@media (prefers-reduced-motion: reduce)` guard exists only
  in `index.html` (L87) and covers `.morph` only. `queues.html` and `case-detail.html` have **no**
  guard for `.seg`/`.tab` transitions or the `showTab` background-flash (`case-detail` L542). Small
  motions, but add the guard. Owner: **stitch-prototyper** / **ui-visual-designer**.
- **[m] Channel column = glyph + `title` only** (`queues.html` L204/213): `✉`/`⌘` on a non-focusable
  `<span title>` — `title` isn't reliably announced and the `⌘` glyph for WhatsApp is opaque. Add
  visually-hidden text. Owner: **stitch-prototyper**.
- **[m] Data grid is divs, not a table** (`queues.html` L194–270): VRM/Case/PO/Provider/Status columns
  have no `role="table"/row/columnheader`, so column↔cell relationships aren't exposed.
- **[m] `prov.none` `#AEB4BF` = 2.08:1** on white (`case-detail` L272) and `spseg` future states 3.21:1
  — conveys "no provenance / upcoming"; low contrast but arguably non-essential. Confirm intent.
- **[m] Error states not wired to AT:** VAT `.ctrl.err` (L271) and the field-9 conflict message (L277)
  have no `aria-invalid`/`aria-describedby` linking the control to its error text.
- **[m] Decorative SVG icons** in rail/header lack `aria-hidden="true"` (named by parent `title`, so
  low risk of double-read).

## Gating verdict

**BLOCKED for convergence.** Four AA blockers (B1 missing headings/`main`, B2 unlabeled form controls,
B3 keyboard-inoperable queue selector, B4 `faint`/amber contrast) each independently breach the AA
floor and all recur across the set. The direction has good bones — a genuine focus ring, shape-coded
badges, and a working keyboard model — so the fixes are mechanical, not structural. Re-audit after B1–B4
land before this direction is allowed to win.
