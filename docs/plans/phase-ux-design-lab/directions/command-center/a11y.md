# a11y audit тАФ `command-center` direction (Graphite NOC)

**Method:** static WCAG-2.1-AA audit of the rendered markup + computed token contrast
(`index.html`, `queues.html`, `case-detail.html`). Contrast ratios computed from the CSS custom
properties (sRGB relative luminance; translucent badge fills blended over `--ground`).

**Verdict: GATED тАФ not convergence-ready.** Strong visual a11y bones (focus halo, reduced-motion,
multi-coded status, high body-text contrast) sit on a thin semantic/operability layer: several
controls are non-focusable `div`/`span` click-handlers (Level-A keyboard failures), there is no
`<main>` and no headings, search/command inputs have no accessible name, and custom tabs/segmented/
grid widgets carry no ARIA roles or state. These are blockers, not polish.

**Score: 2.5 / 5.**

---

## What passes (credit where due)

- **Reduced motion** тАФ all three files ship `@media (prefers-reduced-motion:reduce){*{transition:none!important;animation:none!important}}`; `index.html` also neutralises the refresh-spin. PASS (frontend-design motion floor met).
- **Focus indicator (where present)** тАФ `.btn`, `.gauge`, `.trow`, `.crow`, `.qrow` use a 2-ring `:focus-visible` halo (`0 0 0 2px var(--ground), 0 0 0 3px var(--signal)`); signal-on-ground = 8.16:1, well over the 3:1 non-text floor. Clear and consistent on those elements.
- **Body & status text contrast** тАФ `--tx` 16.1:1, `--tx2` 8.9:1, `--tx-mut` 5.35:1 on `--ground`; status colours on their tinted badge fills 5.4тАУ6.9:1 (review/held/ready/new). All AA.
- **VRM plate** тАФ dark ink on `#FFD400` = 13.9:1; honours "never light-on-yellow."
- **Not colour-alone** тАФ provenance chips are shape (`тЬФ тЧП тЦ▓ тАФ`) + colour + text; required/empty fields are red **and** carry "тЧг required" text + `.ferr` message; severity uses bar-width + duration string, not colour only. Good.
- `lang="en"`, viewport meta, native `<details>`/`<summary>` for the JSON disclosures, and real `<button>` elements for tabs/segments/most actions.

## Failures (located, by severity)

### BLOCKER тАФ keyboard operability (WCAG 2.1.1, Level A)
Interactive controls implemented as non-focusable `div`/`span` with JS click handlers тАФ unreachable by keyboard, no role, no name:
- **Refresh** `<span class="rfs">тЖ╗` тАФ `index.html:380`, `queues.html:243`, `case-detail.html:329`.
- **Modal close** `<span class="x">тЬХ` тАФ `case-detail.html:562`.
- **Reflection toggle** `<label class="switch">` (no `<input>`, no `role="switch"`/`aria-checked`) тАФ `case-detail.html:430`.
- **Toolbar filters** `<div class="filter">Provider/Status/Channel/Age` тАФ `queues.html:267тАУ270` (look like dropdowns, not focusable/operable).
- **Address options** `<div class="addr">` radio-style picker (no `role="radio"`/`radiogroup`, not focusable) тАФ `case-detail.html:447тАУ464`.
- **Readiness "тЖТ field" jumps** `<div class="chk no">` тАФ `case-detail.html:534,536`.
- **"тАж N more тЖТ"** `<div class="tcol-f">` тАФ `index.html:427,444,461`.
- **Cmd-K action items** `<div class="cmdk-it">Refresh board / Toggle density` тАФ `index.html:589тАУ590`.
тЖТ owner: **stitch-prototyper** (promote to `<button>`/`<a>` or add `role`+`tabindex`+key handlers).

### BLOCKER тАФ missing accessible names (WCAG 4.1.2 / 3.3.2, Level A)
- **Search inputs** wrapped in `<label class="search">` whose only text is the `тМХ` glyph + `/` kbd тЖТ computed name is garbage; add `aria-label="Search cases"`. `index.html:374тАУ378`, `queues.html:242`.
- **Cmd-K input** placeholder-only, no name тАФ `index.html:581`, `queues.html:346`.
- **Icon-only buttons** with no text/`aria-label`: ghost `тЖЧ` "open in mailbox" `index.html:413,419,425` (only the first has `title`); modal `тЬХ`; refresh `тЖ╗`.
тЖТ owner: **stitch-prototyper**.

### BLOCKER тАФ structure & landmarks (WCAG 1.3.1 / 2.4.1 / 2.4.6, Level A/AA)
- **No `<main>`** тАФ content column is `<div class="main">` (all three). No main landmark / skip target.
- **No headings at all** тАФ crumbs are `div`, band titles (`.rtitle`) and section heads are `span`; zero `<h1>`тАУ`<h6>` in any page. No heading navigation for SR users.
- **Primary nav is `<aside class="rail">`** (complementary), not `<nav>` тАФ all three. (The mobile `.tabbar` is correctly `<nav>`.)
- **Sections** (`<section class="band">`) have no accessible name (no `aria-labelledby`/heading) тЖТ unnamed region landmarks.
тЖТ owner: **stitch-prototyper** (ux-architect to confirm landmark/heading IA).

### HIGH тАФ custom widgets lack ARIA roles/state (WCAG 4.1.2 / 1.3.1)
- **Case-detail tabs** `.tabs/.tab/.panel` тАФ no `role="tablist|tab|tabpanel"`, `aria-selected`, `aria-controls`; active state is class-only. `case-detail.html:374тАУ382`.
- **Queues segmented control** `.segs/.seg` тАФ no `aria-pressed`/`role`, selected state class-only. `queues.html:249тАУ262`.
- **Queues data grid** `.thead/.qrow` and **case-detail field rows** `.frow` / history `.hist` are `display:grid` `div`s with no `role="grid|row|columnheader|gridcell"` тАФ SR users get no table semantics. `queues.html:285тАУ339`, `case-detail.html:388+,512+`.
- **Row "buttons"** `.trow/.crow/.qrow[tabindex=0]` are focusable but role-less generic divs (Enter navigates) тАФ add `role="link"`/`button`.
тЖТ owner: **stitch-prototyper**.

### HIGH тАФ contrast on informational text (WCAG 1.4.3, fails 4.5:1)
`--tx-dis #515C6E` is used for *informational* microcopy and fails on every surface:
- search **placeholder** on `--surf1` = **2.73:1** (`index.html:376`, `queues.html:242`).
- **`.gatetag`** pills ("box" / "gated" / "not ready") = **2.5тАУ2.9:1** тАФ and the gated buttons carrying them are real `<button>`s **without** `disabled`/`aria-disabled` (`index.html:533,541,557`; `queues.html:58`; `case-detail.html:361тАУ363` тАФ note the *true* Submit at `552/363` correctly uses `disabled`+`title`, so the gated ones are inconsistent).
- `.peak` (gauges), `.ssub` (seg subtitle, `queues.html:251`), `.qhint` (`queues.html:340`), `.pterm` (pipeline), `.gh-note` (R3), `.cmdk-grp` group labels тАФ all `--tx-dis`, all < 3:1.
- Marginal small-text: **Conflict** badge `--st-err` on its fill = **4.24:1**; **Submitted** `--st-sub` = **3.94:1** (11px bold = not "large", so both fail 4.5). `queues.html:304,316`.
тЖТ owner: **ui-visual-designer** (lift `--tx-dis` toward `--tx-mut` тЙИ 5.3:1 for any text that conveys meaning; reserve sub-3:1 only for genuinely disabled controls).

### MEDIUM тАФ touch-target size (prompt floor тЙе44px; WCAG 2.5.8 AA min 24px)
- **`.btn.tiny`** (confirm/reclass/draft/file-req/classify/open) padding `3px 7px`, 11px text тЖТ ~22тАУ24px tall тАФ under 44px; the **icon-only** `тЖЧ`/`тЖ╗`/`тЬХ` (~16тАУ22px) fail even the 24px AA minimum.
- `.facet`/`.filter`/`.chaser-tab` chips ~26тАУ30px. `.chk.no` jump rows ~28px (and non-focusable).
- Mobile (`тЙд767`): row actions only grow to ~30px; tabbar links (54px) are fine.
тЖТ owner: **mobile-ux-designer** (min 44├Ч44 hit area, or 24px + spacing, on touch breakpoints).

### LOW
- Evidence thumbnails `<img alt="">` (`case-detail.html:434тАУ438`) тАФ empty alt is defensible (decorative + role text beside), but the **overview "reg visible"** photo conveys gate-relevant info; give it a real `alt`.
- Decorative glyphs (`.ic`, `.g`, `.vi`, `.pg`, `.dia`) are not `aria-hidden` тЖТ SR may announce symbol names alongside the adjacent text label. Add `aria-hidden="true"`.
- Modal/Cmd-K dialogs have `role="dialog"`+`aria-label` and Escape-to-close, but no `aria-modal`, no focus move-in/restore, no focus trap, and the close affordance is non-focusable (see BLOCKER).
