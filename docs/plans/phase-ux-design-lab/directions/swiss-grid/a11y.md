# a11y audit — swiss-grid direction

**Scope:** static WCAG 2.1/2.2-AA audit of the three runnable mockups — `index.html` (Cockpit),
`queues.html`, `case-detail.html`. Source-level inspection of rendered markup + computed contrast
(no live browser run). Audited by the accessibility engineer for Stage-D convergence gating.

**Verdict:** `CONDITIONAL — not AA-clean as-is.` Strong accessibility bones (visible focus, colour
never carries meaning alone, reduced-motion correctly gated, landmarks present, primary row targets
44 px). Held back by one **systematic contrast failure** (the `ink-400 #9A9DA4` grey used as real
text), **missing heading semantics**, **two unlabeled textareas**, and a **mouse-only submit modal**.
All are mechanical, located fixes — no architectural rework.

**Score: 3 / 5.**

## Findings (located, by severity)

### HIGH — must fix before this direction can gate AA

1. **`ink-400 #9A9DA4` used as informational text everywhere — fails 1.4.3 (2.4–2.7:1).**
   This is the dominant defect; it recurs on all three pages. Located instances:
   - Eyebrow captions overridden to `!text-ink-400`: `index.html` L182 ("Live depth drains…"),
     L268 ("Row → confirm/reclassify…"), L384 ("Drafts only…"); footers L416 / `queues` L346 /
     `case-detail` L499; `case-detail` L373, L397, L493.
   - The `.ord` measure-gutter numerals (`#9A9DA4`) carrying R0–R5 and field numbers 01–12 —
     `case-detail` field gutter on `#FAFAF9` = 2.60 (L215+), queues row ords L189+.
   - `text-ink-400` mono metadata: address-corpus "SEEN 14 · LAST …" (`case-detail` L381/L385),
     pipeline stage numerals (L127–139), "·TODAY/·WK" suffixes, "low confidence" (L245).
   - Owner: **ui-visual-designer** (token choice). Fix: promote informational `ink-400` text to
     `ink-500 #6B6F76` (≥4.5 on white/FAFAF9, 4.57 on paper) or `ink-600 #52555C` (7.5 on white).
     Reserve `ink-400` for non-text hairlines/decoration only.

2. **No heading structure (1.3.1 / 2.4.6); `case-detail.html` has no `h1`.**
   Section titles are `.eyebrow` `<span>`/`<p>`, not headings; every `<section>` is unnamed. A
   screen-reader user gets zero heading navigation. `index`/`queues` have a single `h1` but it is
   `hidden sm:block` (display:none) so it disappears on mobile. `case-detail` exposes the case id
   only as a `<span>` (L116) — no `h1` at all.
   - Owner: **stitch-prototyper**. Fix: make each `ruled-head` eyebrow an `<h2>`/`<h3>` (visually
     unchanged), give each `<section>` `aria-labelledby`, and keep one always-visible `h1` per page.

3. **Unlabeled textareas (4.1.2 / 1.3.1).** `case-detail` Notes textarea (L430) and IBA-reason
   textarea (L394) have placeholder text only — no `<label>`/`aria-label`. Placeholder is not a name.
   - Owner: **stitch-prototyper**. Fix: add `aria-label` (e.g. "Internal note", "IBA reason").

4. **Submit modal is mouse-only (2.1.1 keyboard).** The route-modal is toggled by
   `#submit-toggle` which is `class="hidden"` (`display:none` → not focusable); open/close/cancel are
   all `<label for>` elements, which are not in the tab order and don't respond to Enter. Keyboard
   users cannot open it, and once open (via mouse) cannot Escape/close it. No `role="dialog"`,
   `aria-modal`, labelled name, or focus move. The visible `✕` (L512) has no accessible name.
   - Owner: **stitch-prototyper** (markup) / **fluent-codeapp-designer** (the real port must use a
     focusable trigger + dialog semantics + focus trap + Esc).

### MEDIUM

5. **Operable controls styled as disabled with failing text (1.4.3).** The "Submit to EVA"
   triggers are `<label class="act gated cursor-pointer">` (`case-detail` L168, L478) — genuinely
   operable, but `.act.gated` text is `#9A9DA4` on `#FAFAF9` = 2.60. The 1.4.3 disabled-control
   exemption does **not** apply to an operable element. (Truly `disabled` buttons — Copy JSON,
   Enrich, Open in Box, modal Submit — are exempt and fine.)
   - Owner: **ui-visual-designer** / **stitch-prototyper**.

6. **Tabular data is not a table (1.3.1).** Triage (`index` L213+) and the queues list
   (`queues` L180+) render column headers as `.eyebrow` spans over CSS-grid `<a>` rows — no
   programmatic row/column-header association. Owner: **stitch-prototyper** (add `role="table/row/
   columnheader/cell"` or real `<table>` semantics).

7. **Touch targets below the 44 px lab bar (clear WCAG 2.2 §2.5.8 24 px AA, miss the lab floor).**
   REFRESH `h-9`=36 (L123 etc.), `.tab`=40, `.filt`=36, `.chip`=28, sub-nav `h-8`=32 (`index` L77),
   mobile bottom-nav ≈40 px, address `.act`/radio chrome 34. Primary targets are fine (rail/rows
   44–48 px). The modal `✕` and raw native radios/checkboxes are the only items near the 24 px AA
   floor. Owner: **mobile-ux-designer**.

8. **Duplicate `nav` landmarks unnamed (1.3.1).** Sidebar `<nav>` and mobile `<nav>` on each page
   have no `aria-label` to distinguish them. Owner: **stitch-prototyper**.

### LOW

9. Decorative SVG search icons (`index` L117, `queues` L114/158) lack `aria-hidden="true"`
   (sparkline SVGs correctly set it — `index` L318+). 10. Low-contrast placeholder text doubles as the
   only visible field hint (search `aria-label` is just "Search"; the typeable hint "VRM · Case/PO ·
   claimant" is 2.72:1). 11. `tab.gated`/"Enrich · gated" reads as inactive but is non-interactive
   text at 1.94:1 — keep it, just darken.

## Confirmed PASSES (strengths — do not regress)

- **Visible focus indicator:** global `:focus-visible{outline:2px solid #1D3FD6;outline-offset:1px}`
  on a/button (+ input/textarea on case-detail), plus `.focusable:focus-within` for wrapped fields.
  Ring contrast 6.96–7.68 vs adjacent surfaces — passes 1.4.11. (2.4.7 PASS.)
- **Colour is never the sole signal (1.4.1):** status badges are glyph **+** UPPERCASE word **+**
  colour (e.g. `▲ CONFLICT`, `▣ INSTRUCTION`, `▤ MISSING IMAGES`); severity rows pair the border
  colour with `OVERDUE`/`DUE` wording; readiness uses `✗`/`✓` glyphs. Shape-coded and colour-coded.
- **Reduced motion (2.3.3/animation):** every transition is gated behind
  `@media (prefers-reduced-motion:no-preference)` — reduced-motion users get none. No keyframe/auto
  animation. Correct direction. PASS.
- **Landmarks:** `aside`/`header`/`main`/`nav`/`footer` on every page; `aria-current="page"` on
  active nav; `lang="en"`; zoomable viewport (no `maximum-scale`).
- **Named inputs:** search, filter, mileage (L268), sequence (L522) all carry `aria-label`.
- **Primary hit targets** (rail items, all row links) are 44–48 px.

## Top fixes (ordered)

1. Retire `ink-400 #9A9DA4` as a text colour → `ink-500`/`ink-600`; fixes ~30 instances across all
   three pages in one token change. *(ui-visual-designer)*
2. Add heading semantics: promote `ruled-head` titles to `h2/h3`, `aria-labelledby` each `section`,
   and one always-visible `h1` per page incl. `case-detail`. *(stitch-prototyper)*
3. `aria-label` the two textareas (Notes, IBA reason). *(stitch-prototyper)*
4. Make the submit modal keyboard-operable — focusable trigger, `role="dialog"`+`aria-modal`,
   named `✕`, Esc-to-close, focus move/trap. *(stitch-prototyper → fluent-codeapp-designer for port)*
5. Fix the operable-but-`.gated` Submit labels' text contrast (≥4.5). *(ui-visual-designer)*
6. Give the data lists table semantics; name the duplicate `nav`s; `aria-hidden` the decorative
   search icons. *(stitch-prototyper)*
7. Raise secondary controls (REFRESH, tabs, filter chips, mobile-nav, sub-nav) toward the 44 px lab
   target. *(mobile-ux-designer)*

**Gating note for design-critic:** treat this direction as **AA-blocked** on findings 1–4 until
remediated. No keyboard trap and focus is always visible, so it is not a hard barrier — but it is not
AA-clean and should not win convergence unscored on a11y.
