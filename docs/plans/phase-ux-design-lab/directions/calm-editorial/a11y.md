# A11y scorecard — calm-editorial ("The Reading Room")

**Method:** static WCAG-2.1/2.2-AA audit of the rendered markup + computed CSS for
`index.html`, `queues.html`, `case-detail.html`. Contrast ratios computed from the
frozen token palette (sRGB, WCAG formula). No browser run; findings are located by
file + selector.

**Verdict:** CONDITIONAL — strong a11y bones (focus, landmarks, reduced motion,
colour-not-alone), but **multiple real AA failures** block a clean pass. Not AA-clean
as it stands.

**Score: 3 / 5.**

---

## What passes (keep)

- **Focus indicator** — global `:focus-visible{outline:2px solid var(--accent); outline-offset:2px}`
  covers `a, button, summary, [tabindex], input, select, textarea, .tile`. Ring contrast
  accent vs paper **8.35:1**, vs surface **8.87:1** — well over the 3:1 non-text floor. Best-in-class.
- **Landmarks / structure** — `nav[aria-label="Primary"]`, `main`, `header`, `section[aria-labelledby]`
  with real `<h2>`s; `aria-current="page"` on active nav. Pipeline hero is `role="img"` + descriptive
  `aria-label`, inner decoration `aria-hidden`.
- **Reduced motion** — all three pages ship `@media (prefers-reduced-motion: reduce){*{transition-duration:0ms!important;animation:none!important}}`. The one flagged motion moment (refresh spin) is a CSS transition → suppressed.
- **Colour-not-alone** — status carries label + shape (▲/●/✓ `.pm` marks) + tone; due pills say
  "overdue/due"; dup VRM carries `sr-only` "duplicate suspected". Disciplined.
- **Primary text contrast** — ink **14.99:1**, ink-2 **8.28:1**, ink-muted **5.61:1**; status text
  review/held/ready on their tints **4.96 / 4.93 / 5.45:1** all pass AA.
- **sr-only provenance** — every provenance dot/icon has an `sr-only` gloss ("source PDF, reviewed");
  decorative glyphs (carets, dots, flags, ⌕ key) are `aria-hidden`. Inputs have `aria-label`s.
- `lang="en-GB"`, page `<title>`s, zoomable viewport (no `user-scalable=no`).

## Failures (located, by severity)

### HIGH — `--ink-faint #97A1B0` used as real text (systemic 1.4.3)
Contrast: on paper **2.46:1**, on surface **2.61:1**, on sunken **2.30:1** — all far under 4.5
(and under 3:1 even as large text). Not decorative — it labels meaningful content:
- `index.html`: `.cstage .cowner` (System/Us/Them owner labels), the **entire R3 windowed-throughput
  block** (`#r3h` heading + `.note` + `.wstat .eyebrow` + `.wsub` — "In today / Submitted today /
  Cleared this week" and their captions), `.masthead-kick` faint eyebrow, search `::placeholder`.
- `queues.html`: `.qtab .qc` tab counts (9/12/8/4), `.fchip .n` reason counts, `.qrow .po.none`,
  the trailing explainer `.meta` ("One case lives in exactly one queue…").
- `case-detail.html`: `.sstage .slabel` pipeline-spine stage labels (New/Parsing/Chasing/Ready/
  Submitted/Box), `.field .num` EVA field numbers 1–12, `.gatehint`, `.facts .note`.
> Owner: **ui-visual-designer** — raise the faint token (≈ `#6B7policy…` → target ≥ `#71798A` gives
> ~4.5 on paper) or restrict `ink-faint` to genuinely decorative use and promote any text role to
> `ink-muted` (already passing at 5.6:1).

### HIGH — keyboard operability of core controls (2.1.1 / 4.1.2)
- **Whole-row navigation, no keyboard activation.** `.qrow` (queues) and `.wrow` (cockpit Chase-next)
  carry `tabindex="0"` + `onclick="location.href='case-detail.html'"` but are `<div>`s with no
  `keydown`/Enter handler and no inner link. Keyboard users can focus the row but **cannot open the
  case** — the primary affordance is mouse-only.
- **Exclude toggle not operable.** `.switch` is a `<label>` wrapping decorative spans, toggled by a
  JS `click` only — not focusable, no `role="switch"`/`aria-checked`. Excluding a reflection photo
  (a real EVA gating action) is unreachable by keyboard and silent to SR.
- **Address picker has no semantics.** `.addr` rows are `tabindex="0"` `<div>`s; `.radio` is
  `aria-hidden`; "picked" is visual only — no `role="radiogroup"/radio`, no `aria-checked`, no
  keyboard selection. SR users can't tell or change which inspection address is selected.
> Owner: **stitch-prototyper** (make rows real `<a>`/`<button>` or add Enter/Space + role;
> give `.switch` a real `<input type=checkbox>` / `role="switch"`+`aria-checked`; model `.addr`
> as a radiogroup). Keyboard-reorder of evidence is claimed in copy but not wired.

### MEDIUM-HIGH — touch / target size (2.5.8)
- `.tact` inline row actions (Confirm / Reclassify / Open in mailbox / Jump to case / Classify /
  Dismiss) are `border:0; padding:0`, ~**18px** tall — under the 24×24 AA minimum, in a multi-button
  cluster (gap 14px). Fails 2.5.8 and is far from the 44px comfort target.
- Below 44px but ≥24px (AA-pass, sub-ideal): `.refresh` 32×32, `.filter` 36px, `.fchip` 30px,
  `.btn` 38px, chaser `.sel` 38px.
> Owner: **mobile-ux-designer** — give `.tact` a ≥24px (ideally 44px) hit area via padding/min-height;
> review the 30–38px controls against the 44px target for the touch port.

### MEDIUM — broken ARIA table (1.3.1) — `queues.html`
`<div role="table" aria-label="Review queue">` contains `.qrow` divs with **no** `role="row"`,
`role="cell"`, or `role="columnheader"`, and the visible `.grid-head` is `aria-hidden="true"`.
SR announces a table with zero rows and no header association.
> Owner: **stitch-prototyper** — use a real `<table>` (or `role="rowgroup"/row/cell/columnheader`
> and drop `aria-hidden` on the head), or remove `role="table"` and expose rows as labelled links.

### MEDIUM — unlabeled / under-described controls (3.3.2 / 4.1.2)
- `case-detail.html` IBA `<textarea>` (`.iba textarea`) has only a `placeholder`, **no label /
  aria-label** — unnamed field. (The chaser `.draft` and the two `.sel`s are correctly `aria-label`led.)
- Gated `disabled` buttons (Open in Box / Enrich / Box File-Request) are fine, but the only gated cue
  for SR is the panel/`gatehint` text; the 🔒 is `aria-hidden`. Acceptable, noted.
> Owner: **stitch-prototyper** (add a label to the IBA textarea).

### MEDIUM-LOW — non-text contrast of control boundaries (1.4.11)
White/paper inputs and buttons are delineated only by a 1px `--divider #D2D8DF` border =
**1.44:1** on surface / **1.35:1** on paper (and `--hairline` **1.25:1** on `.addr`/`.thumb`) —
under the 3:1 needed to identify a control's boundary. Affects `.search`, `.tb-search`, `.filter`,
`.sel`, `.draft`, `.vrm` plate, `.refresh`, `.fchip`, `.addr`, `.thumb`.
> Owner: **ui-visual-designer** — darken the control-border token (or add a fill delta) to ≥3:1.

### LOW — heading & tab-pattern conformance (1.3.1 / 2.4.6, APG)
- `case-detail.html` has **no `<h1>`**; the case identity (VRM / Case-PO / vehicle) is spans/divs,
  and the only headings are the gated-panel `.h2`s → heading order starts at h2. Add an `<h1>`
  (sr-only is fine).
- Tabs (`.qtabs`, `.tabs`) use `role="tablist"/tab"`+`aria-selected` but **no `aria-controls`,
  panels lack `aria-labelledby`/`tabindex`, and there's no Left/Right roving-tabindex** (APG tabs).
  Operable but not the full pattern. In `queues.html` the Ready tab sits inside a `<span class="spacer">`
  — a `role="tab"` that isn't a direct child of the tablist.
> Owner: **ux-architect** (pattern) + **stitch-prototyper** (wire `aria-controls`/`labelledby`, add h1).

## Gating verdict (for convergence)

**CONDITIONAL — flag, do not let win on a11y as-is.** No single catastrophic blocker, but the
direction carries **two HIGH AA failures** (systemic `ink-faint` text contrast; keyboard-dead core
controls — row-open, photo-exclude, address-pick) plus a touch-target and a broken-table fail. The
foundation (focus ring, landmarks, reduced motion, colour-not-alone, sr-only provenance) is genuinely
strong, so these are fixable without a redesign. Clears to AA once the HIGH/MEDIUM items above are
resolved by their owners.

## Top fixes (priority order)
1. Stop using `--ink-faint` for text — raise the token or promote text roles to `ink-muted`. *(ui-visual-designer)*
2. Make row-open, the exclude toggle, and the address picker keyboard-operable with real roles/`aria-checked`. *(stitch-prototyper)*
3. Give `.tact` row actions a ≥24px (target 44px) hit area. *(mobile-ux-designer)*
4. Fix or remove `role="table"` in queues; label the IBA textarea; add an `<h1>` to case-detail. *(stitch-prototyper)*
5. Lift control-border contrast to ≥3:1 (1.4.11). *(ui-visual-designer)*
