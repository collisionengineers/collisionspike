# A11y audit — Soft-Pack Bento direction

**Scope:** static WCAG-2.1/2.2-AA audit of the three runnable mockups in this folder
(`index.html`, `queues.html`, `case-detail.html`). Method: source read + contrast math on the
frozen token palette (sRGB relative-luminance, 4.5:1 normal / 3:1 large-text & UI-component floor).
No browser run — pure static review.

**Verdict: CONDITIONAL PASS / FLAGGED — 3.5 / 5.** Strong accessibility floor (landmarks, skip
link, a genuinely visible focus ring with a contrasting halo, a `prefers-reduced-motion` path,
most form controls labelled, status that is shape+colour+text rather than colour-alone, gated
features rendered truly `disabled` not faked). It is **flagged for convergence**: it carries
**unresolved AA contrast failures on enabled primary CTAs**, a **search field with no accessible
name**, an **incomplete ARIA tab pattern**, **no modal focus management**, and **a few sub-24px
touch targets**. None are catastrophic; all are concrete and fixable. This direction may compete,
but must clear the Serious items below before it could *win* a converged round.

---

## What's already right (floor met)

- **Landmarks + skip link** on every page: `<a href="#main">Skip to content</a>`, `<aside>` rail
  with `<nav aria-label="Primary">`, `<main id="main">`, `<header>`, `<footer>`, `<h1>` per page,
  sections with `aria-labelledby`. Heading order is sane (h1→h2→h3, no skips).
- **Visible focus indicator**: `:focus-visible { box-shadow: 0 0 0 2px #EDEBE6, 0 0 0 4px #5B5BD6 }`
  applied to `a/button/input/select/textarea/summary/[tabindex]`. The 2px iris ring measures
  **4.51:1 vs the tray bg and 5.37:1 vs white** — clears the 3:1 non-text-contrast floor (1.4.11 /
  2.4.11), with a halo gap so it reads on coloured fills too.
- **Reduced motion**: all three carry `@media (prefers-reduced-motion: reduce){*{animation:none
  !important;transition:none!important}}`; `index.html` additionally neutralises `scroll-behavior`
  and the `.lift` hover transform.
- **Not colour-alone**: every status carries a glyph **and** a word (`▲ Review`, `◆ Held`,
  `⚠ Conflict`); due-severity uses bar-height + colour + text (`3d over` / `today` / `2d`); the
  required field shows tint + border + the word "required". Good.
- **Gated = disabled, not faked**: Box / Enrich / Submit / Sentry-REST render with the real
  `disabled` attribute and an explanatory `title` — correct, and exempt from contrast minimums.
- **Most controls are labelled**: VAT select, photo-role selects, address radios, IBA-reason input,
  chaser textarea, sequence input, the exclude toggles (`role="switch"` + `aria-checked` +
  `aria-label`), modal close, hamburger — all have accessible names.

## Located failures (by severity)

### Serious — AA contrast on enabled, interactive text

Computed ratios (white = `#FFFFFF`):

| Pair | Ratio | Need | Where |
|---|---|---|---|
| white on `apricot #F97316` | **2.80** | 4.5 | `index.html:483` "Draft" CTA (enabled `<a>`); and **every `.act.primary` rest state** (`case-detail.html` CSS `:99`) — currently only saved by `disabled`, will fail the moment Submit/primary is enabled |
| white on `teal #0D9488` | **3.74** | 4.5 | `index.html:338` "→ Case" row CTA (enabled; hover→teal-ink 5.47 passes, rest fails) |
| white on `review #F43F5E` | **3.67** | 4.5 | `queues.html` `.facet[aria-pressed=true]` (`:100`, e.g. "Missing images" `:221`); rail count pills `bg-review/90 text-white` ("9"/"8", 11.5px bold) on all 3 pages |

These are normal-size text (11.5–13px) so the 4.5:1 floor applies — the 3:1 large-text exception
does **not** rescue them. → **ui-visual-designer**: darken the fills to their `*-ink` token under
white text, or switch to dark-text-on-tint (the page already proves both patterns elsewhere).

### Serious — missing accessible name

- **Search input has no name.** `index.html:233` / `queues.html:170`:
  `<input id="q" placeholder="VRM · Case/PO · claimant">` sits inside a `<label>` that contains only
  an icon and a "/" hint — **placeholder is not a label** (3.3.2 / 4.1.2). Add `aria-label="Search"`
  (or visible label text). → **stitch-prototyper**.

### Moderate

- **Incomplete ARIA tab pattern** (`case-detail.html:254–262`): `role="tablist"` + `role="tab"` +
  `aria-selected` are present, but tabs have **no `aria-controls`**, panels have **no
  `role="tabpanel"`/`aria-labelledby`**, and there is **no arrow-key roving-tabindex** (click-only).
  Announced as tabs but not operable as the WAI-ARIA tab model. → **stitch-prototyper**.
- **Modal focus management absent** (`case-detail.html:481–531`, JS `:554–557`): opening the S5
  dialog does **not move focus into it, does not trap focus, does not restore focus on close**, and
  the background is not inert. Esc-close and backdrop-close work, and it is `display:none` when
  shut (good), but a keyboard/SR user can tab out into the page behind it (2.4.3). → **stitch-prototyper**.
- **Non-interactive `<div>`s styled as controls** (`queues.html:210–213`): the Provider / Status /
  Channel / Age "`.fdrop`" filter dropdowns are bare `<div>`s — not focusable, no `role`, no
  keyboard handler. They read as actionable but are inert. → **stitch-prototyper** (make `<button>`
  with `aria-haspopup`, even in the mock).
- **Touch targets below 24px** (WCAG 2.2 §2.5.8 minimum; well below the lab's 44px aspiration):
  - exclude switches `.switch` are **38×22px** (22 < 24) — `case-detail.html` CSS `:129`, used `:339/349/359`.
  - native radios `w-4 h-4` = **16px** with no wrapping `<label>` to extend the hit area — `case-detail.html:370/378/386`.
  - standalone inline `Fix →` / deep-link buttons (~16–18px tall) in the readiness rail — `case-detail.html:448–450`.
  → **mobile-ux-designer**: pad to ≥24px (ideally 44px), and wrap radios in their label.

### Minor

- **Decorative glyphs / emoji not hidden from AT.** Rail nav icon spans (`▤ ◆ ▦ ⬓ ⚙ ⛭ ◇`, all
  pages) and the provenance emoji (`📄 ✦ 🛡 📚`, `case-detail.html:268–299`) are not
  `aria-hidden="true"`, so SRs announce them ("black diamond", "shield") ahead of the real label.
  Add `aria-hidden`. → **stitch-prototyper**.
- **Marginal tab/badge contrast.** `apricot-ink on apricot-tint = 4.47` and `held-ink on
  held-tint = 4.49` at **11px** (small text → needs 4.5) — fail by a hair on the "Ready"/"Held"/
  "Chasing-Held" tabs and amber badges. (The amber treatment is *directionally* correct —
  dark-on-amber, never white-on-amber, which white-on-`#F59E0B`=2.15 would catastrophically fail.)
  Nudge the ink one step darker. → **ui-visual-designer**.
- **Low-contrast greys on the spine / field index** (`case-detail.html`): spine "ghost" labels
  `#B7BCC9` = **1.90**, empty-bin `#8A8FA0` = 3.22, field index numbers `#9AA0AE` = **2.62** —
  informational (EVA field order 1–12, pipeline position), so worth lifting above 4.5. → **ui-visual-designer**.
- **Smooth-scroll ignores reduced-motion in JS.** `case-detail.html:547` hardcodes
  `scrollIntoView({behavior:'smooth'})`; `queues.html`/`case-detail.html` reduce-blocks don't reset
  `scroll-behavior`. Gate on `matchMedia('(prefers-reduced-motion: reduce)')`. → **stitch-prototyper**.
- `aria-disabled="true"` items (the "Engineer · reserved" nav link, the "Enrichment ⊘" tab) remain
  focusable/clickable — prefer real `disabled` or `tabindex=-1` + click guard.

## Convergence gating verdict

**FLAGGED — not a clean AA pass.** Eligible to compete, but blocked from *winning* until the
**Serious** items clear: (1) white-on-apricot / white-on-teal / white-on-review enabled-CTA
contrast, and (2) the unnamed search field. The **Moderate** items (tab pattern, modal focus,
inert `.fdrop` divs, sub-24px targets) should land before this ports to production. Routing:
contrast + grey tokens → **ui-visual-designer**; markup/ARIA/search-name/tab-pattern/modal-focus/
decorative-emoji → **stitch-prototyper**; touch targets → **mobile-ux-designer**.
