# A11y scorecard вҖ” `dataviz-forward` В· "Telemetry Deck"

> Static WCAG-2.1/2.2-AA audit of the three runnable mockups
> (`index.html`, `queues.html`, `case-detail.html`) in this folder. Source-level review of the
> *rendered* markup + CSS + the computed palette contrast. Audited 2026-06-25.
> Owner routing per file boundaries: **ui-visual-designer** (colour), **stitch-prototyper**
> (markup/ARIA), **mobile-ux-designer** (touch targets).

## Verdict

**CONDITIONAL PASS вҖ” score 3 / 5.** Strong, accessibility-aware foundation (consistent visible
focus, full `prefers-reduced-motion`, status never by colour alone, good landmarks/lang/zoom). But
several **genuine AA blockers** remain, concentrated in keyboard operability of `<div>`-based
controls, unlabelled inputs/controls, the collapsed-nav icon names, touch-target sizing, and two
contrast misses. All are localised fixes, not a redesign вҖ” **gate convergence on the four SERIOUS
items below before this direction is allowed to win.**

### Contrast FAILS

- **`idle #6B7A91` as small status text = 4.0вҖ“4.4:1 (FAIL, needs 4.5).** Affects the `NOT READY` /
  `INGESTED` / idle status badges (`.st-idle`, `queues.html:107`, `index.html:241`) and the
  `UNIDENTIFIED В· NEEDS HUMAN` chip (`.chip.unid`, `index.html:164`). The large idle numerals
  (`.qn`, `.pseg .pn` 30вҖ“34px) clear the 3:1 large-text bar and are fine. вҶ’ **ui-visual-designer:**
  lift idle to вүҲ`#8294AB`+ for text use, or only use idle at large-text sizes.
- **Primary CTA `#fff` on `accent #3B82F6` = 3.68:1 (FAIL, needs 4.5).** `.btn.primary`
  (`case-detail.html:62`) вҖ” the "Submit to EVA" button. Masked in the mock because it renders
  `:disabled` (ink3-on-raised, 4.59:1 вҖ” passes), but the **enabled** design state fails. вҶ’
  **ui-visual-designer:** darken the fill (e.g. `accent-p #2563EB` = ~5.0:1) or use ink-dark text.
- **Transient hover: `danger` text on `--raised` = 4.24:1 (FAIL on hover).** Row hover
  (`.trow/.qrow/.crow:hover{background:raised}`, e.g. `queues.html:95`) drops danger-toned badge
  text and `.out b` below 4.5. Moderate (hover-only). вҶ’ **ui-visual-designer.**

## Visible focus вҖ” PASS (strength)

All three files define `:focus-visible{outline:2px solid var(--focus);outline-offset:2px;border-radius:3px}`
(`index.html:32`, `queues.html:24`, `case-detail.html:24`). Ring contrast 7.9вҖ“8.6:1, offset so it
clears the control edge. Applies to every real `<button>`/`<a href>`. **Caveat:** it only helps
elements that can receive focus вҖ” the `<div>` controls below never do (see Keyboard).

## Reduced motion вҖ” PASS (strength)

`@media (prefers-reduced-motion:reduce){*{animation:none!important;transition:none!important}}` in
all three (`index.html:284`, `queues.html:153`, `case-detail.html:235`). Freezes the Flow-Channel
shimmer, the `.live` pulse, dwell-tick build, and the JS refresh colour-flash. Matches direction В§5.

## Colour-not-alone вҖ” PASS (strength)

Status is glyph + UPPERCASE label + colour everywhere (`.stbadge .g` + text; provenance badges carry
`src` + UPPERCASE + shape glyph; dwell track always paired with a mono `Nd` label appended in JS).
Satisfies SC 1.4.1 across the board. **One exception:** the queue **Channel** column conveys
email-vs-webhook by icon shape alone (`вңү`/`вҢ¬`, `title` only вҖ” `queues.html:251,260,269`) with no
text equivalent.

## Keyboard operability & ARIA вҖ” the SERIOUS gaps

1. **[SERIOUS] Address picker is keyboard-inoperable.** `.addrcard` is a clickable `<div>` with a JS
   click handler and no `tabindex`/`role` (`case-detail.html:441вҖ“455`, handler `583`). Picking the
   inspection address is a *core* task ("staff pick or edit") and cannot be done without a mouse, and
   shows no focus ring. вҶ’ **stitch-prototyper:** make each option a `<button>` or radio in a
   `radiogroup`. 
2. **[SERIOUS] Collapsed / tablet nav items have no accessible name.** At вүӨ1279px the text label is
   hidden (`.nav a .lbl{display:none}` вҖ” `index.html:249`, `queues.html:125`, `case-detail.html:215`),
   leaving only decorative Unicode glyphs (`в–Ј вңү в–Ө в—Ҳ вӨ“ вҡҷ вҢ¬`) as the link content вҖ” i.e. unlabelled
   links on tablet. вҶ’ **stitch-prototyper:** add `aria-label` to each nav `<a>` (and the bottom-bar
   links), `aria-hidden` on the glyph.
3. **[SERIOUS] Search inputs have no programmatic label.** `<label class="search">` wraps only the
   `вҢ•` glyph + the `<input>`; placeholder is not an accessible name (`index.html:316вҖ“318`,
   `queues.html:221`). Accessible name resolves to the glyph char or empty. вҶ’ **stitch-prototyper:**
   add `aria-label="Search cases"` (and `aria-hidden` on `вҢ•`).
4. **[SERIOUS] IBA reason input has no label.** `<input placeholder="Reason for IBAвҖҰ">`
   (`case-detail.html:459`) вҖ” placeholder-only, and the value is *required* before IBA fallback.
   вҶ’ **stitch-prototyper:** real `<label for>`.

### Moderate ARIA / semantics

- **Filter dropdowns are non-operable `<div>`s.** `.fsel` Provider/Status/Channel/Age
  (`queues.html:222вҖ“225`) look like selects but have no role, tabindex, or handler вҖ” unreachable by
  keyboard. вҶ’ **stitch-prototyper:** real `<button>`/`<select>` (even in the mock).
- **Toggle state not exposed.** Segment tabs (`.segtab` `index.html:388`), queue partitions/facets
  (`.pseg`/`.facet` `queues.html:185,232`), and case-detail `.tab` (`case-detail.html:310`) signal
  active state by `.on` class only вҖ” no `aria-pressed`/`aria-selected`/`aria-controls`. They are real
  `<button>`s (keyboard-operable вҖ” good) but a SR user can't tell which is active. вҶ’ **stitch-prototyper.**
- **Case-detail tabs aren't a tab widget.** `.tab`/`.tabpane` lack `role="tablist|tab|tabpanel"` and
  arrow-key handling (`case-detail.html:310вҖ“318`). Minimum operability is met (Tab+Enter); upgrade to
  the ARIA tabs pattern. вҶ’ **stitch-prototyper.**
- **Queue "grid" has no table semantics.** `.ghead`/`.qrow` are `<div>`/`<span>` streams
  (`queues.html:241+`); the column headers (VRM/Case/Status/Outstanding/Age) aren't programmatically
  associated with cells. вҶ’ **stitch-prototyper:** `role="table/row/columnheader/cell"` or a real `<table>`.
- **No page `<h1>` on queues / case-detail.** Page title is a `<span class="pgtitle">`
  (`queues.html:176`) / breadcrumb span (`case-detail.html:258`); sidebar jumps to `<h3>`. index.html
  is fine (`<h1>` + `<h2>`s). вҶ’ **stitch-prototyper:** promote the page title to `<h1>`.
- **Decorative glyphs read aloud.** Status/flag glyphs (`в–І вң” в—Ҹ в—Ӣ в—Ҷ вҡ‘ вҳ…`) and the dup `вҡ‘` / channel
  `вңүвҢ¬` aren't `aria-hidden`; SRs announce their Unicode names as noise (the meaning is in the adjacent
  text, so not a blocker). вҶ’ **stitch-prototyper:** `aria-hidden="true"` on all decorative glyph spans.
- **Two `<nav>`s, no distinguishing label**, and no skip-link (landmarks present, so 2.4.1 is met via
  regions вҖ” skip-link is a recommendation). вҶ’ **stitch-prototyper.**

## Touch targets вҖ” moderate

The direction promises **вүҘ44px** (В§1.1, В§6) and WCAG 2.2 SC 2.5.8 floors at 24Г—24px.

- **`.btn.sm` вүҲ 22вҖ“24px tall** (padding 5px/10px, 11.5px text вҖ” `index.html:84`, `case-detail.html:56`):
  the triage **Confirm / Reclassify / Open / Classify / Link-to-Case** actions, **Copy draft**,
  **Log as chaser**. Borderline on the 24px AA floor, **fail the spec's own 44px bar**, and they do
  **not** enlarge at the phone breakpoint. вҶ’ **mobile-ux-designer.**
- Other interactive elements sit 28вҖ“36px (`.btn` ~30, `.facet` ~24, `.tmplbtn` ~28, `.segtab` ~36,
  `.tab` ~35, nav `~31`): clear 24px, under 44px. The mobile bottom-bar links are correct
  (`min-height:48px`). вҶ’ **mobile-ux-designer:** raise in-content controls toward 44px (or вүҘ44px hit
  area via padding) for the touch breakpoints.

## Top fixes (priority order)

1. **(stitch-prototyper)** Make the **address picker** real controls (radio/`<button>`) вҖ” keyboard +
   focus. *Core task, currently mouse-only.*
2. **(stitch-prototyper)** **Label every input/control**: `aria-label` on search + nav-collapsed
   links + bottom-bar; `<label>` on the IBA reason input; make `.fsel` filters real controls.
3. **(ui-visual-designer)** Fix **contrast**: lift `idle` for text use (4.0вҶ’вүҘ4.5), darken the
   **Submit-to-EVA** primary fill or use dark text (`#fff`@3.68вҶ’вүҘ4.5), and the danger-on-`raised`
   hover (4.24вҶ’вүҘ4.5).
4. **(mobile-ux-designer)** Grow **`.btn.sm`** and other in-content controls to the promised **44px**
   hit area, including at the phone breakpoint.
5. **(stitch-prototyper)** Expose **state/semantics**: `aria-pressed`/`aria-selected` on
   segment/partition/facet/tab toggles; table (or grid) roles on the queue grid; `<h1>` on
   queues/case-detail; `aria-hidden` on decorative glyphs.
