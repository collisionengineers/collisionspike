# Accessibility audit — Brutalist Utility ("Concrete Ledger")

Static WCAG 2.1/2.2-AA audit of the three runnable mockups in this directory. Source read only; no
browser render. Contrast ratios computed from the CSS token hexes. CE-brand a11y rules
(`docs/design/THEME-MAPPING.md`: focus halo, blocker pills, **amber never white-on-amber**, shape+colour
badges) applied where relevant.

- **Files:** `index.html` (cockpit), `queues.html` (review ledger), `case-detail.html` (case)
- **Score: 2.5 / 5** · **Gating verdict: FAILS AA — CONDITIONAL (blocked from convergence until the 3 blockers clear)**
- **Audited:** 2026-06-25

---

## Verdict

The direction has a **solid a11y floor** — one global visible focus ring, a `prefers-reduced-motion`
path on every page, `lang="en"`, real `role="dialog"`/`role="status"`, and status that is **glyph + text,
not colour-alone** (satisfies the CE shape+colour rule). But three independent **AA blockers** stop it
winning convergence as-is: a systemic white-on-orange contrast failure, a fully keyboard-inoperable queue
table, and a total absence of headings. None are design-fatal; all are mechanical fixes routed below.

---

## Blockers (must fix before this direction can converge)

### B1 — White-on-orange held status is ~3.6:1 (AA 1.4.3 fail + CE amber-rule violation) · owner: ui-visual-designer
`--orange:#E8590C` with `#fff` text computes to **3.58:1** — below the 4.5:1 floor for the text size used,
and a direct breach of the frozen CE rule *"amber never white-on-amber (dark-on-amber 8–9:1)."* It is
**systemic** (every "Held/Chasing" surface):
- `index.html:124-126` `.station.held` + `:341` (pipeline held station), and the peach subtext
  `.station.held .ss{color:#FFE3D2}` (`:125`) is **~3.07:1** — worse.
- `index.html:217` `.stamp.held`; `queues.html:88` `.stamp.held`; `case-detail.html:79` `.stamp.held` /
  spine `.snode.held` (`case-detail.html:95`).
- `index.html:522` snapbar `HELD` span (white on orange).
Also orange-as-text on light: `index.html:140` `.tcol.other …::after "· NEEDS A HUMAN"` (orange on
`#FBEFE9`) = **~3.34:1**, fail.
**Fix:** switch held surfaces to dark-ink text on amber (target 8–9:1), or darken the amber. Recolour the
peach subtext and the orange caption to ink.

### B2 — Queue rows are mouse-only (AA 2.1.1 Keyboard fail) · owner: stitch-prototyper
`queues.html:260,269,278,287,296,305,314,323` — every ledger row is `<tr onclick="location.href=…">` with
**no `tabindex`, no role, no focusable child**. The row cells contain only static spans, so a keyboard
user cannot reach or activate any case in the Review ledger — the primary work surface is inoperable.
**Fix:** make the case identifier a real `<a href>` (row-spanning link), or give the row
`role="link" tabindex="0"` + Enter/Space handler + the global focus ring.

### B3 — No headings anywhere (AA 1.3.1 / 2.4.6) · owner: stitch-prototyper
None of the three pages contain a single `<h1>`–`<h6>`. Every title is a styled `<div>`/`<span>`
(`.rt` `index.html:334` etc., `.cluster-h` `case-detail.html:135`, `.side-h` `:209`). No page H1, no
heading-navigation for screen-reader users. Section `aria-label`s create *region* landmarks but do not
replace headings (and R0–R5 each as a region is landmark noise).
**Fix:** add a visually-hidden (or styled) `<h1>` per page and promote region/cluster titles to `<h2>/<h3>`.

## Minor / prototype-acceptable

- **Icon-only modal close** `✕` has no accessible name (`case-detail.html:559`) — reads as a glyph. Add `aria-label="Close"`. (stitch-prototyper)
- **`case-detail.html` has no `<main>` landmark** — primary content sits in `<div class="main">` (the `<header>` becomes a banner and `<aside aria-label="Readiness">` is complementary, but no `main`). (stitch-prototyper)
- **`.tab.gat` gated-tab text uses `--border-mid` (#8A8A84) = 3.48:1 on white** (`case-detail.html:129`, "Enrichment") — fails 4.5 for 12px bold. Same colour as text also fails for the search placeholder and crumb separators, and the `aria-hidden` snapbar "NOT READY" white-on-#8A8A84 (3.48:1). (ui-visual-designer)
- **Decorative glyphs not hidden** — rail/tab/stamp icons (`▣ ✉ ▤ ★ ▲` etc.) are not `aria-hidden="true"`; names survive via adjacent `.lbl`/text, but SR output is noisy. (stitch-prototyper)
- **Fake form controls** — `.sel-in` "MILES ▾"/"choose ▾" are `<span>`, not `<select>` (`case-detail.html:384-385`): not keyboard-operable, unlabelled. The IBA reason `<input>` (`:464`) and chaser `<textarea>` (`:479`) lack labels. (The submit `.editseq` *is* `aria-label`led — good, `:566`.) Production-only concern, but flag. (stitch-prototyper / fluent-codeapp-designer at port)
- **Gated controls** mix real `<button>` with no `aria-disabled` (`case-detail.html:342-344`) and inert `<span class="btn gated">` (`:493,580`); "NOT CONNECTED" text carries the state, so acceptable, but prefer `aria-disabled`. (stitch-prototyper)
- **Focus ring on the dark rail** — cobalt `#1A38E5` ring vs `#111` rail ≈ 2.51:1, under the 3:1 non-text floor for the indicator on unselected rail links. Fine on paper/white surfaces (6–7.7:1). (ui-visual-designer)
- **Content photos `alt=""`** (`case-detail.html:425+`) — defensible for placeholder previews (role/flags described in text); real vehicle photos need meaningful alt at the port. (stitch-prototyper)

## What passes (keep)

- **Visible focus**: `:focus-visible{outline:3px solid var(--cobalt);outline-offset:2px}` global on all 3 pages (`index.html:37`, `queues.html:35`, `case-detail.html:34`). Cobalt = 6.3:1 vs paper, 7.7:1 vs white — passes the 3:1 indicator floor on light surfaces.
- **Reduced motion**: `@media (prefers-reduced-motion:reduce){*{transition:none !important}}` on all 3 (`index.html:282`, `queues.html:182`, `case-detail.html:278`). Only motion present is 0.08s transitions, fully covered.
- **Not colour-alone**: status stamps, provenance badges, photo flags, and thermo ages all pair colour with a **text label + glyph** (e.g. `Needs review ▲`, `EXCLUDE · PERSON`, `2d04h`) — meets CE shape+colour rule and 1.4.1.
- **Core contrast is strong**: ink `#0A0A0A` on paper = 16.3:1; `--ink-muted #57574F` on white/paper/surface-2 = 6.1–7.4:1 (passes); VRM ink-on-`#FFD400` plate = 13.8:1; white-on-red `#E10600` = 4.98:1 (pass); white-on-green = 5.47:1; white-on-cobalt = 7.7:1.
- **Landmarks/labels present**: `<nav aria-label="Primary">`, `<main>` (index + queues), `<aside aria-label="Readiness">`, `role="dialog" aria-modal="true"`, `role="status"`, search `aria-label="Search"`, `aria-current="page"` on active rail link.
