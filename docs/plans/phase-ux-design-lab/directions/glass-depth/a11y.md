# Accessibility audit — `glass-depth` (Aurora Glass) direction

**Scope:** static WCAG-2.1/2.2-AA review of the three runnable mockups
(`index.html`, `queues.html`, `case-detail.html`) — colour contrast, visible focus,
semantic structure/landmarks, ARIA/labels on controls & icons, touch-target sizing,
reduced motion. Source-level audit (rendered-DOM spot checks where needed).

**Verdict:** **Conditional pass — gate BLOCKED for convergence until the High-severity
items land.** Unusually strong a11y foundation (focus-visible on everything, *both*
`prefers-reduced-motion` and `prefers-reduced-transparency` handled, shape+colour status
coding, dark-ink tokens that clear AA on solid white), but it ships **two Level-A keyboard
failures** (mouse-only primary navigation + mouse-only photo controls) and a **thin
landmark/heading skeleton**. These are mechanical to fix and do not require a redesign.

**Score: 3 / 5.**

---

## Severity legend
- **HIGH** — Level-A / gating; blocks convergence.
- **MED** — Level-AA failure or AA-borderline; fix before production port.
- **LOW** — hygiene / SR-noise; cheap polish.

---

## HIGH — must fix before this direction can win the gate

### H1 · Primary navigation is mouse-only — `queues.html` (SC 2.1.1, 4.1.2, 2.4.3)
Case rows are `<tr onclick="location.href='case-detail.html'">` with **no `tabindex`,
no `role`, no key handler, and no `<a>` inside**. The row is the *only* way into a case —
keyboard and screen-reader users cannot open any case. No focus style exists on rows
(only `:hover`).
**Fix:** make the VRM (or Case/PO) cell a real `<a href>` (semantic + focusable + inherits
the global focus ring), or give the row `tabindex="0" role="link"` + Enter/Space handler.
**Owner:** stitch-prototyper.

### H2 · Photo-exclude toggles are non-operable `<span>` — `case-detail.html` (SC 2.1.1, 4.1.2)
`.sw` exclude switches are `<span>` with a click handler — not focusable, no
`role="switch"`/`aria-checked`, no keyboard. "Exclude (reflection)" is a load-bearing
control (it changes image-rule readiness). Also 30×17 px (target-size fail, see M3).
**Fix:** native `<button role="switch" aria-checked>` or styled `<input type="checkbox">`
with an associated `<label>`.
**Owner:** stitch-prototyper (markup) / mobile-ux-designer (size).

### H3 · No `<main>` landmark; section titles aren't headings — all three (SC 1.3.1, 2.4.1)
`.main` is a `<div>`, not `<main>`. Region titles ("R1 Inbox triage", "R4 Chase next",
"Readiness") are styled `.eyebrow`/`.ch` divs, not `<h2>/<h3>`. `case-detail.html` has
**no `<h1>` at all** (breadcrumb span only; headings start at the dialog `<h2>` / sidebar
`<h3>`). SR users get no skip-to-main and almost no heading map.
**Fix:** wrap content in `<main>`; promote each panel eyebrow to a heading and add a
visually-hidden/visible `<h1>` per page (case-detail: the plate+Case/PO line).
**Owner:** stitch-prototyper; IA confirm by ux-architect.

### H4 · Icon-only rail loses its accessible name at the 1279 px breakpoint — all three (SC 2.4.4, 4.1.2)
At `max-width:1279px` the rail collapses and `.lbl` becomes `display:none`, leaving each
nav `<a>` named only by a decorative glyph (`◆ ✉ ▤ ⬇ ⚙`) or a count pill. Accessible name
degrades to e.g. "black diamond suit" / "8". (Mobile keeps `.lbl`, so this is the tablet
band specifically.)
**Fix:** put a persistent `aria-label` on every nav `<a>` (and `aria-hidden="true"` on the
`.ic` glyph) so the name survives the label being hidden.
**Owner:** stitch-prototyper.

## MED — AA failures / borderline, fix before production

- **M1 · Translucent panels can't guarantee AA for muted small text (SC 1.4.3).**
  `--glass-panel:rgba(255,255,255,.74)` sits over a vivid aurora gradient, so the effective
  background luminance varies. `--ink-muted #4E5474` clears ~7.3:1 on *solid* white but the
  margin erodes to ~AA-or-below over the bright purple/blue regions for the 10–10.5px text
  (`.seg .sub`, `.tile .dr`, `.eyebrow .meta`). The `prefers-reduced-transparency` fallback
  (solid `#fff`) is a good mitigation but only fires for that minority setting.
  **Fix:** raise panel opacity (≥.85) under text, or darken `--ink-muted`, or sit small
  metadata on the solid `--glass-inset`. Verify rendered contrast in the bright-aurora zone.
  **Owner:** ui-visual-designer.

- **M2 · Ghost pipeline tiles dim real numbers below AA (SC 1.4.3).** `index.html`
  `.seg.ghost{opacity:.46}` carries live counts (Submitted **24**, Box **24**). 0.46 opacity
  roughly halves contrast → fails 4.5:1. **Fix:** convey "tally/terminal" via label or weight,
  not opacity; keep numbers ≥4.5:1. **Owner:** ui-visual-designer.

- **M3 · Target sizes below the 24px AA minimum / 44px design floor (SC 2.5.8).**
  `.btn.sm` ≈22px tall (Confirm, Reclassify, Draft, Open, Copy draft, Log as drafted),
  `.sw` 17px, address radios ~13px. Most chrome (`.btn`, `.nav a`, `.drop`, `.segchip`,
  `.facet`, `.tab`) lands ~30–32px — under the 44px floor. **Fix:** min-height 44px (or ≥24px
  with spacing) on interactive elements; enlarge `.sw` and radio hit-areas.
  **Owner:** mobile-ux-designer.

- **M4 · Tabs aren't a real tab pattern — `case-detail.html` (SC 4.1.2).** `.tab` are
  `<button>` (operable by Tab+Enter, good) but the `role="tablist"` has no `role="tab"`,
  `aria-selected`, or `aria-controls`; panes have no `role="tabpanel"`/`aria-labelledby`; no
  arrow-key roving. **Fix:** complete the ARIA tab pattern. **Owner:** stitch-prototyper.

- **M5 · "Disabled" buttons aren't announced disabled (SC 4.1.2).** `.btn.disabled`
  (Open in Box, Enrich, Submit-to-EVA, Drag-to-EVA) is a styling class only — still
  focusable/clickable, no `disabled`/`aria-disabled`, state conveyed by opacity + `title`.
  **Fix:** add `aria-disabled="true"` (keep Submit's intentional modal-open behaviour) so the
  gated state is programmatic, not visual-only. **Owner:** stitch-prototyper / fluent-codeapp-designer (port).

- **M6 · Facet filters expose state by colour only (SC 1.4.1, 4.1.2).** `queues.html`
  review-reason `.facet` vs `.facet.off` differ only by red-tint vs grey; no `aria-pressed`.
  **Fix:** `aria-pressed` toggle (the partition buttons already do this correctly — copy that).
  **Owner:** stitch-prototyper.

- **M7 · Form controls without programmatic names (SC 1.3.1, 4.1.2).** The VAT `<select
  class="sel-inline">` and the Image-Based-Assessment reason `<input type="text">` have only a
  nearby `.fl` div / placeholder — no `<label for>`/`aria-label`. **Fix:** associate labels.
  **Owner:** stitch-prototyper.

- **M8 · Duplicate-risk flag is colour+glyph with no text (SC 1.4.1).** `.plate.dup` red ring
  + `.flag ▲` signals a duplicate with no accessible text. **Fix:** add visually-hidden
  "duplicate risk" text / `aria-label` on the flag. **Owner:** stitch-prototyper.

- **M9 · Submit modal lacks focus management (SC 2.4.3, 2.1.2).** `.scrim role="dialog"
  aria-modal="true"` does not move focus in on open, has no focus trap, and no `Esc`-to-close
  (Cancel is mouse `onclick`). **Fix:** focus the dialog on open, trap Tab, close on `Esc`,
  restore focus to the trigger. **Owner:** stitch-prototyper (mock) / fluent-codeapp-designer (port).

- **M10 · Channel/type colour-dots carry meaning alone (SC 1.4.1).** `.dotk` (receiving /
  query / other) and the `.ch` channel envelope rely on colour/glyph; `.ch` has a `title` but
  the dot has nothing. **Fix:** add SR text for the category. **Owner:** stitch-prototyper.

## LOW — hygiene / SR-noise

- **L1** Decorative glyphs not hidden from SR (nav `.ic`, `.arrow →`, status `▲ ◆ ▸ ◦`,
  ribbon legend swatches) — read as Unicode names. Add `aria-hidden="true"`.
- **L2** `queues.html` `<th>` missing `scope="col"`.
- **L3** `case-detail.html` breadcrumb is a bare `<span>`; wrap in `<nav aria-label="Breadcrumb">`.
- **L4** `index.html` "↻ Refresh" is a `<span class="rf">` (cursor:pointer) — not a button;
  not keyboard-operable. Make it a `<button>`.
- **L5** No `aria-live` on the "Updated 09:42" / ember status so dynamic changes are silent
  (cosmetic in the mock).

## Strengths (carry these into the port)
- **Focus visible everywhere:** global `:focus-visible{outline:2px solid var(--accent);
  outline-offset:2px}` — every native control inherits a clear ring. (Verify the purple ring
  keeps ≥3:1 on the dark rail; it is ~2.8:1 there — nudge to a lighter token on the rail.)
- **Motion + transparency both respected:** `prefers-reduced-motion: reduce` kills all
  animation/transition; `prefers-reduced-transparency: reduce` swaps to solid panels and drops
  `backdrop-filter`. This is the accessibility floor met deliberately.
- **Not colour-alone for status:** badges pair a shape glyph + word (`▲ Review`, `◆ Held`,
  `▸ Ready`); readiness uses check/cross + text; provenance uses glyph+colour+`title`.
- **Good ARIA already present:** `aria-current="page"` on active nav, `aria-pressed` on the
  queue partition buttons, `aria-label` on sections/search/address-radios, `aria-modal` on the
  dialog, real `<table>` semantics in queues, search input labelled + magnifier `aria-hidden`.
- **Colour tokens are dark and high-contrast on solid surfaces:** `--ink #161A2E` ~15:1,
  `--ink-2 #3A4060` ~10:1, `--accent #7C3AED` ~5.9:1, UK plate `#0A0A0A`-on-`#FFD400` ~17:1.
- `lang="en-GB"` set; zoomable viewport (no `maximum-scale` lock).

## Gating summary
| Area | State |
|---|---|
| Colour contrast | Mostly AA on solid; **at-risk over translucent aurora (M1) + ghost tiles (M2)** |
| Visible focus | **Strong** (global focus-visible); rail-ring ~2.8:1 to lift |
| Keyboard operability | **FAIL** — H1 rows, H2 photo toggles |
| Landmarks / headings | **FAIL** — H3 no `<main>`, no `<h1>` on case-detail |
| ARIA / labels | Partial — H4 icon-rail, M4 tabs, M5 disabled, M7 inputs |
| Target size | **Below floor** — M3 |
| Reduced motion / transparency | **PASS** |

**Convergence verdict:** do not let `glass-depth` win until **H1–H4** are resolved
(all mechanical). With those fixed it is a credible AA contender; M1–M3 should follow
before the Fluent v9 port.
