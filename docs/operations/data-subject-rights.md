# Data-subject rights

This procedure covers access, rectification, erasure, restriction, objection, and portability requests.
It is operator/legal work and requires an approved request identity and scope.

## Locate

Search by the minimum verified identifiers across PostgreSQL, source evidence, transient storage, the
Archive, mail references, File Request links, and external identifier strings. Preserve an audit of each
search and avoid widening to unrelated cases.

## Access and portability

Export the structured case record, field-source history, audit, and relevant source evidence in a secure
package. Explain derived/suggested values separately from source facts.

## Rectification

Correct the authoritative relational record through the normal audited route. Because the Archive is a
one-way copy, record whether a corrected copy must be added there; never silently replace source
evidence.

## Erasure

Check the legal/evidential hold first. A valid hold pauses the request and records the reason. Otherwise
follow the approved retention policy across every located store. Archive deletion is manual only. Record
what was removed, anonymized, retained, or could not be changed, with the legal basis for each outcome.

## Close-out

Have a second authorized person verify the inventory and result, send the approved response, retain the
minimum compliance record, and update the owning ticket. Do not treat a database-only action as complete
cross-store erasure.
