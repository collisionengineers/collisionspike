# Azure resource overlap, duplication & waste (#3)

This review examines the live `rg-collisionspike-dev` footprint and the Function/OCR Bicep templates for overlapping, duplicated, or wasted Azure resources. Most of what looks like "duplication" turns out to be either intentional, gated-off forward provisioning or a known, deliberately-deferred ALM cut — only a small set of genuine drift/hygiene items remain, and none have material dollar impact in a near-idle FC1/F0 dev sandbox. All findings are analysis-only; nothing was deleted.

Confirmed findings by severity: **0 critical, 0 high, 3 medium, 1 low, 1 info** (6 total confirmed; 2 dropped during verification).
