# Dead code, bloat & time bombs — Code App frontend (#7)

This section catalogues the verified dead code, bloat, and latent "time bomb" findings in the Power Apps Code App frontend (`mockup-app/`). Every item below is a confirmed finding (the verifier independently re-read the cited code and upheld the factual core), though several had their severity corrected downward because the originally-flagged behaviour is mock-by-design, gated off, or already documented as pending activation work. The dominant theme is that the live deployed app is still a mock for its core review workflow: edit handlers, EVA submission, dedup resolution, and evidence-byte persistence all operate on local React state or fire toasts, while a substantial built-but-unwired layer (canonical EVA serializer, finalize transport, write-side adapters, decision tables) sits orphaned beside it.

Confirmed findings by severity: 0 critical, 0 high (all three originally-"high" items were corrected to low/medium during verification), 3 medium, 5 low, 1 info — **9 confirmed total**.
