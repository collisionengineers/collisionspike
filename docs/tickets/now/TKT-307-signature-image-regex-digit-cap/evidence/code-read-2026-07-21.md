Both cap the digit run at four, so `image078315.png` (a real six-digit Outlook cid filename)
matches neither. `_is_image_evidence_file` / `_isImageEvidenceFilename` then fall through to the
extension/hint tiers and read it as genuine image evidence.

Materialization: the Python regex is authored once and vendored by
`scripts/build/sync-engine.py` into `services/functions/parser/` and `services/functions/ocr/`
(confirmed identical pre-fix via `check-engine-materialized.py`). The TS twin is a separate,
hand-maintained file — not covered by the sync tooling — kept parallel by comment only.

A separate mechanism, the area floor (`AREA_FLOOR = 200*200` in `image-sniff.ts`;
`_MIN_EXTRACTED_IMAGE_AREA = 200*200` in `service.py`), filters by raster pixel area, not
filename — it does not catch a full-size cid attachment and is unaffected by this fix.
